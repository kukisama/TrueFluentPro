# 配置中心 3.4 集成设计

> 编制日期：2026-03-04
> 目标：将当前独立弹窗 `ConfigCenterView` 集成到主窗体 `SettingsView` 中，消除模态对话框交互。

---

## 1. 现状分析

### 1.1 当前架构

| 组件 | 形式 | 行数 | 说明 |
|------|------|------|------|
| `ConfigCenterView.axaml` | 独立 Window（模态对话框） | 621 行 | 包含 9 个 TabItem，左侧 Tab 导航 |
| `ConfigCenterView.axaml.cs` | 代码后置（事件驱动） | 1,317 行 | 大量手动控件操作、无 ViewModel |
| `SettingsView.axaml` | 主窗体内嵌 UserControl | 91 行 | 简易仪表板，含"打开配置中心"按钮 |

### 1.2 ConfigCenter 9 个 Tab

| # | Header | 内容概述 |
|---|--------|----------|
| 1 | 订阅与鉴权 | Azure 订阅 CRUD、密钥管理、区域选择、连接测试 |
| 2 | 录音与存储 | 录音格式、Azure Storage Blob 配置与验证 |
| 3 | AI 基础配置 | AI Provider 选择、API Key/Endpoint、AAD 认证 |
| 4 | 输入与识别 | 语音识别参数（语言、置信度、超时） |
| 5 | 字幕与文本 | 实时文本长度、历史条目数、字号 |
| 6 | 洞察 | AI 洞察预设按钮、系统提示词 |
| 7 | 复盘 | 复盘 Sheet 预设与自定义提示词 |
| 8 | Media Studio | 图片/视频生成 AI 配置（独立 Provider） |
| 9 | 实时翻译字号 | 默认字号选择 |

### 1.3 问题

1. **模态对话框阻断工作流** — 打开配置中心后无法同时操作翻译/复盘
2. **代码后置过重** — 1,317 行事件驱动代码，无 ViewModel，不易维护
3. **与 SettingsView 重复** — 订阅选择、设备管理在两处分别实现
4. **保存语义不清晰** — 用户必须点"保存"按钮，否则修改丢失

---

## 2. 目标设计

### 2.1 总体方案

将 ConfigCenter 所有功能直接嵌入 `SettingsView`，取消弹窗。SettingsView 使用 **分组卡片 + 可展开面板** 的形式替代 TabControl，整体为单页滚动布局。

### 2.2 SettingsView 新结构

```
SettingsView (ScrollViewer)
├── 订阅管理卡片
│   ├── 当前订阅选择（合并现有 SettingsView 功能）
│   ├── 订阅列表 + CRUD（来自 ConfigCenter Tab 1）
│   └── 连接测试按钮
├── 音频设备卡片（保留现有 SettingsView 功能）
├── 录音与存储卡片（来自 ConfigCenter Tab 2）
├── AI 基础配置卡片（来自 ConfigCenter Tab 3）
├── 语音识别卡片（来自 ConfigCenter Tab 4）
├── 字幕与文本卡片（来自 ConfigCenter Tab 5 + 9）
├── AI 洞察预设卡片（来自 ConfigCenter Tab 6）
├── 复盘预设卡片（来自 ConfigCenter Tab 7）
├── Media Studio 配置卡片（来自 ConfigCenter Tab 8）
├── 外部链接卡片（保留现有 SettingsView 功能）
└── 帮助与关于卡片（保留现有 SettingsView 功能）
```

### 2.3 卡片组件设计

每个卡片使用统一的 `Expander` + `Border` 组合：

```xml
<Border BorderBrush="#FFE5E7EB" BorderThickness="1" CornerRadius="8" Padding="16">
    <Expander Header="订阅管理" IsExpanded="True">
        <!-- 卡片内容 -->
    </Expander>
</Border>
```

### 2.4 保存策略

- **即时保存**：每个字段变更后自动保存（通过 debounce 防抖）
- 移除全局"保存"按钮
- 保留"测试连接"等验证按钮
- 状态栏显示"配置已自动保存"提示

## 3. ViewModel 改造

### 3.1 新增 `SettingsViewModel`

从 `ConfigCenterView.axaml.cs` 的 1,317 行事件驱动代码中，提取业务逻辑到 `SettingsViewModel`：

```
SettingsViewModel
├── SubscriptionManagement（订阅 CRUD + 验证）
├── StorageConfig（录音与 Blob 存储）
├── AiProviderConfig（AI 基础配置）
├── RecognitionConfig（输入与识别参数）
├── TextDisplayConfig（字幕与文本 + 字号）
├── InsightPresetsConfig（洞察预设管理）
├── ReviewPresetsConfig（复盘预设管理）
├── MediaStudioConfig（Media Studio AI 配置）
└── AutoSave（防抖自动保存逻辑）
```

### 3.2 DI 注册

```csharp
services.AddSingleton<SettingsViewModel>();
```

### 3.3 与 ConfigViewModel 关系

- `ConfigViewModel` 保留为轻量运行时订阅状态管理（选中订阅、语言切换）
- `SettingsViewModel` 管理完整配置编辑（CRUD、验证、高级参数）
- 两者共享同一 `AzureSpeechConfig` 实例

## 5. 风险与注意事项

| 风险 | 缓解措施 |
|------|----------|
| SettingsView 内容过多导致滚动过长 | 使用 Expander 默认折叠非常用区域 |
| 自动保存频繁写磁盘 | 使用 500ms debounce + dirty flag |
| ConfigCenterView 代码后置复杂 | 分步提取，每步验证构建 |
| 已有 ConfigCenter 测试流程失效 | 保持功能等价，逐项对照验证 |

- AI Config View（`AiConfigView.axaml`）保持独立——它是洞察面板的入口
