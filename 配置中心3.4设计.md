# 配置中心 3.4 集成设计

> 编制日期：2026-03-04
> 目标：将当前独立弹窗 `ConfigCenterView` 集成到主窗体 `SettingsView` 中，消除模态对话框交互。

---

## 1. 现状分析

### 1.1 当前架构

| 组件 | 形式 | 行数 | 说明 |
|------|------|------|------|
| `ConfigCenterView.axaml` | 独立 Window（模态对话框） | 621 行 | 包含 9 个 TabItem，左侧 Tab 导航 |
| `ConfigCenterView.axaml.cs` | 代码后置（事件驱动） | 1,317 行 | 大量手动控件操作、无 ViewModel |
| `SettingsView.axaml` | 主窗体内嵌 UserControl | 91 行 | 简易仪表板，含"打开配置中心"按钮 |

### 1.2 ConfigCenter 9 个 Tab

| # | Header | 内容概述 |
|---|--------|----------|
| 1 | 订阅与鉴权 | Azure 订阅 CRUD、密钥管理、区域选择、连接测试 |
| 2 | 录音与存储 | 录音格式、Azure Storage Blob 配置与验证 |
| 3 | AI 基础配置 | AI Provider 选择、API Key/Endpoint、AAD 认证 |
| 4 | 输入与识别 | 语音识别参数（语言、置信度、超时） |
| 5 | 字幕与文本 | 实时文本长度、历史条目数、字号 |
| 6 | 洞察 | AI 洞察预设按钮、系统提示词 |
| 7 | 复盘 | 复盘 Sheet 预设与自定义提示词 |
| 8 | Media Studio | 图片/视频生成 AI 配置（独立 Provider） |
| 9 | 实时翻译字号 | 默认字号选择 |

### 1.3 当前 AI 配置数据模型

| 类 | 管理范围 | 终结点字段 |
|---|---------|------------|
| `AiConfig` | 文字对话/洞察/复盘 | ApiEndpoint, ApiKey, ProviderType, AAD 四件套 |
| `MediaGenConfig` | 图片生成 | ImageApiEndpoint, ImageApiKey, ImageProviderType, AAD 四件套 |
| `MediaGenConfig` | 视频生成 | VideoApiEndpoint, VideoApiKey, VideoProviderType, AAD 四件套 |

三套终结点配置各自独立，高度冗余。`VideoUseImageEndpoint` 是唯一的复用手段，且仅限图片→视频方向。

### 1.4 问题

1. **模态对话框阻断工作流** — 打开配置中心后无法同时操作翻译/复盘
2. **代码后置过重** — 1,317 行事件驱动代码，无 ViewModel，不易维护
3. **与 SettingsView 重复** — 订阅选择、设备管理在两处分别实现
4. **保存语义不清晰** — 用户必须点"保存"按钮，否则修改丢失
5. **AI 终结点配置冗余** — 文字/图片/视频各维护一套 Endpoint+Key+Auth，同一个终结点要配三遍
6. **模型与终结点耦合** — 模型名散落在各处硬编码，无法统一管理或复用

## 2. 目标设计

### 2.1 总体方案

将 ConfigCenter 所有功能直接嵌入 `SettingsView`，取消弹窗。SettingsView 采用 **左侧锚点导航 + 右侧瀑布流** 布局（ScrollSpy 模式），类似迅雷设置页的交互风格。

### 2.4 左侧导航交互规则

- **点击跳转**：点击左侧导航项 → 右侧 `ScrollIntoView()` 滚动到对应分区
- **滚动高亮**：右侧滚动时 → 检测当前可见分区 → 自动高亮左侧对应导航项
- **固定不滚动**：左侧导航栏随右侧内容固定，不随右侧滚动
- **视觉样式**：当前活动项使用蓝色高亮条（类似迅雷设置页效果）

### 2.9 AI 终结点管理区设计（核心新增）

#### 2.9.1 设计理念

将原来分散在 `AiConfig`（文字）和 `MediaGenConfig`（图片/视频）中的 3 套终结点配置，统一为 **终结点注册表** 模式：

- 用户可以注册 1~N 个 AI 终结点
- 每个终结点下挂载 1~N 个模型，每个模型标记支持的能力（文字/图片/视频）
- 在洞察/复盘/图片/视频等功能分区，通过下拉选择 **终结点+模型** 组合，且仅显示具备对应能力的模型

## 3. AI 终结点数据模型设计

### 3.1 新增模型类

```csharp
/// <summary>模型能力标签（可组合）</summary>
[Flags]
public enum ModelCapability
{
    Text  = 1,   // 文字对话：洞察、复盘、快问
    Image = 2,   // 图片生成
    Video = 4    // 视频生成
}

/// <summary>一个 AI 终结点（Provider 实例）</summary>
public class AiEndpoint
{
    public string Id { get; set; } = Guid.NewGuid().ToString();
    public string Name { get; set; } = "";                        // "t99e", "Azure East US"
    public bool IsEnabled { get; set; } = true;

    // --- 连接信息 ---
    public AiProviderType ProviderType { get; set; } = AiProviderType.OpenAiCompatible;
    public string BaseUrl { get; set; } = "";
    public string ApiKey { get; set; } = "";
    public string ApiVersion { get; set; } = "";                   // Azure OpenAI 专用

    // --- 认证 ---
    public AzureAuthMode AuthMode { get; set; } = AzureAuthMode.ApiKey;
    public string AzureTenantId { get; set; } = "";
    public string AzureClientId { get; set; } = "";

    // --- 模型列表 ---
    public List<AiModelEntry> Models { get; set; } = new();
}

/// <summary>终结点下的一个模型定义</summary>
public class AiModelEntry
{
    public string ModelId { get; set; } = "";                      // "gpt-4o", "dall-e-3"
    public string DisplayName { get; set; } = "";                  // 可选友好名
    public string DeploymentName { get; set; } = "";               // Azure 部署名
    public string GroupName { get; set; } = "";                    // 分组: "GPT", "DeepSeek"
    public ModelCapability Capabilities { get; set; }              // Text | Image | Video
}

/// <summary>功能分区对某个模型的引用（终结点ID + 模型ID）</summary>
public class ModelReference
{
    public string EndpointId { get; set; } = "";
    public string ModelId { get; set; } = "";
}
```

### 3.3 模型查询辅助

```csharp
/// <summary>从所有启用的终结点中，按能力筛选可用模型</summary>
public IEnumerable<(AiEndpoint Endpoint, AiModelEntry Model)>
    GetAvailableModels(ModelCapability required)
{
    return Endpoints
        .Where(ep => ep.IsEnabled)
        .SelectMany(ep => ep.Models
            .Where(m => m.Capabilities.HasFlag(required))
            .Select(m => (ep, m)));
}

/// <summary>解析 ModelReference 为可用的终结点+模型</summary>
public (AiEndpoint? Endpoint, AiModelEntry? Model)
    ResolveModel(ModelReference? reference)
{
    if (reference == null) return (null, null);
    var ep = Endpoints.FirstOrDefault(e => e.Id == reference.EndpointId);
    var model = ep?.Models.FirstOrDefault(m => m.ModelId == reference.ModelId);
    return (ep, model);
}
```

### 3.4 向后兼容（配置迁移）

首次加载旧配置时自动迁移：

1. 检测到旧 `AiConfig.ApiEndpoint` 非空 → 创建终结点 "默认文字"
2. 检测到旧 `MediaGenConfig.ImageApiEndpoint` 非空 → 创建终结点 "默认图片"（或合并到同 URL 终结点）
3. 检测到旧 `MediaGenConfig.VideoApiEndpoint` 非空 → 创建终结点 "默认视频"（或合并）
4. 自动生成 `ModelReference` 指向迁移后的终结点+模型
5. 迁移完成后删除旧字段

## 6. 风险与注意事项

| 风险 | 缓解措施 |
|------|----------|
| SettingsView 内容过多导致滚动过长 | 左侧锚点导航快速跳转，不依赖滚动浏览全部 |
| 自动保存频繁写磁盘 | 使用 500ms debounce + dirty flag |
| ConfigCenterView 代码后置复杂 | 分步提取，每步验证构建 |
| 已有 ConfigCenter 测试流程失效 | 保持功能等价，逐项对照验证 |
| 旧配置迁移失败导致数据丢失 | 迁移前自动备份 config.json；迁移逻辑有单元测试覆盖 |
| 终结点被删除但仍有 ModelReference 指向它 | 删除前检测引用，提示用户重新分配；悬空引用降级为"未配置" |
| 模型能力标记错误（如文字模型标了图片） | 终结点检测时可选 "探测模型能力"（调用 /models 接口） |
