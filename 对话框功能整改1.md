# 对话框功能整改1 — 实施记录

> 基于 `对话框功能.md` 分析文档，对 TrueFluentPro 对话功能进行全面整改。
> 本次整改遵循"独立解耦模块"设计原则，所有对话相关核心组件可独立打包为 DLL。

---

## 一、架构设计：独立解耦模块

### 1.1 模块边界

整改后的对话模块位于 `Controls/Markdown/` 命名空间下，包含以下组件：

| 文件 | 职责 | DLL 就绪 |
|------|------|----------|
| `IDialogMessage.cs` | 消息接口抽象 —— 解耦模块与宿主应用 | ✅ |
| `MarkdownRenderer.cs` | Markdown AST → Avalonia 控件树渲染 | ✅ |
| `CodeHighlighter.cs` | 24+ 语言语法高亮 | ✅ |
| `MarkdownTheme.cs` | 排版参数/字体/色彩配置 | ✅ |
| `SelectionManager.cs` | 跨块文本选择 | ✅ |
| `SmoothStreamingAnimator.cs` | 字符级流式动画 | ✅ |
| `DialogExporter.cs` | 对话导出（Markdown/TXT/JSON） | ✅ |
| `DialogSearchEngine.cs` | 全文搜索引擎 | ✅ |

### 1.2 解耦策略

- **`IDialogMessage` 接口**：定义消息的 Role、Text、ReasoningText、Timestamp、ContentType、MediaPaths、PromptTokens、CompletionTokens 等属性，模块内部只依赖此接口，不依赖 `ChatMessageViewModel` 等宿主类型。
- **`ChatMessageViewModel` 实现 `IDialogMessage`**：通过接口实现桥接宿主应用与模块。
- **事件驱动解耦**：如对话分支（Fork）通过 `ForkRequested` 事件通知宿主，模块不关心分支的实际创建。
- **零宿主依赖**：`DialogExporter`、`DialogSearchEngine` 仅依赖 `IDialogMessage`，可直接迁移。

### 1.3 未来 DLL 打包路径

```
TrueFluentPro.Dialog.dll
├─ Controls.Markdown.IDialogMessage
├─ Controls.Markdown.MarkdownRenderer
├─ Controls.Markdown.CodeHighlighter
├─ Controls.Markdown.MarkdownTheme
├─ Controls.Markdown.SelectionManager
├─ Controls.Markdown.SmoothStreamingAnimator
├─ Controls.Markdown.DialogExporter
└─ Controls.Markdown.DialogSearchEngine
```

## 二、P0 核心体验提升

### 2.1 ✅ LaTeX 数学公式渲染

**改动文件：** `Controls/Markdown/MarkdownRenderer.cs`

**实施内容：**
- 在 Markdig 管道中启用 `.UseMathematics()` 扩展
- 新增 `MathBlock` 块级渲染（`$$...$$`）：居中显示，等宽字体，斜体，LaTeX 标签，圆角背景
- 新增 `MathInline` 行内渲染（`$...$`）：等宽字体，斜体，`CodeInlineForegroundKey` 配色
- `MathBlock` 继承自 `CodeBlock`，在 switch 分发中需置于 `FencedCodeBlock` 之前

**渲染效果：**
- 块级公式以居中的等宽斜体文本显示，带 "LaTeX" 标签和背景色
- 行内公式以 pill 样式显示，与行内代码风格一致但使用斜体区分

### 2.2 ✅ 消息编辑与重发

**改动文件：**
- `ViewModels/MediaSessionViewModel.cs` — `EditMessageCommand` + `EditMessage()` 方法
- `Views/MediaStudioView.axaml` — 用户消息区增加 ✏ 编辑按钮

**实施内容：**
- 用户消息气泡底部新增"✏ 编辑"按钮
- 点击后将消息文本加载到 `PromptText` 输入框
- 截断该消息及其后续所有消息（编辑后可重新发送）
- 仅对用户消息（`IsUser`）且非流式状态可用

### 2.3 ✅ 对话搜索

**改动文件：**
- `Controls/Markdown/DialogSearchEngine.cs` — 全新搜索引擎
- `ViewModels/MediaSessionViewModel.cs` — 搜索命令 + 状态属性
- `Views/MediaStudioView.axaml` — 搜索栏 UI
- `Views/MediaStudioView.axaml.cs` — Ctrl+F 快捷键 + 事件处理

**实施内容：**
- **Ctrl+F** 快捷键触发搜索栏显示/隐藏
- 搜索栏位于聊天区顶部，包含：搜索输入框、匹配计数、上/下导航按钮、关闭按钮
- `DialogSearchEngine` 对 `_allMessages` 进行大小写不敏感的全文搜索
- 搜索范围覆盖主文本和推理文本
- 支持循环导航（到末尾后跳回开头）

## 三、P1 功能增强

### 3.1 ✅ 对话导出

**改动文件：**
- `Controls/Markdown/DialogExporter.cs` — 全新导出器
- `ViewModels/MediaSessionViewModel.cs` — `ExportConversationCommand`
- `Views/MediaStudioView.axaml` — 📤 导出按钮

**实施内容：**
- 支持三种导出格式：**Markdown**（完整格式，含推理过程、Token 用量）、**纯文本**、**JSON**
- 通过 Avalonia StorageProvider 打开保存文件对话框
- 根据文件扩展名自动选择导出格式
- 导出内容包含：会话名称、导出时间、消息角色/时间/内容/推理过程/Token 用量/附件

### 3.2 ✅ 代码块行号

**改动文件：** `Controls/Markdown/MarkdownRenderer.cs`

**实施内容：**
- 新增 `ShowCodeLineNumbers` 依赖属性（可通过 XAML 绑定到设置）
- 启用行号时使用 Grid 布局：左列灰色行号 + 右列代码内容
- 行号列宽度自适应（1-9行 24px / 10-99行 32px / 100+行 42px）
- 行号与代码之间有透明度分隔线

### 3.3 ✅ Markdown 图片嵌入

**改动文件：** `Controls/Markdown/MarkdownRenderer.cs`

**实施内容：**
- 在 `RenderInlines` 中检测 `LinkInline.IsImage`
- 在段落级通过 `ContainsImage()` 检测图片，使用 `RenderParagraphWithImages()` 拆分渲染
- 图片控件使用 Avalonia `Image`，限制 MaxWidth=600 / MaxHeight=400
- **异步加载**：支持 HTTP URL 和本地文件路径
- 加载状态：显示"🖼 加载中..."占位文本，失败显示"⚠ 图片加载失败"
- alt 文本显示在图片下方（斜体、小字）

### 3.4 ✅ 对话分支（Fork）

**改动文件：**
- `ViewModels/MediaSessionViewModel.cs` — `ForkFromMessageCommand` + `ForkRequested` 事件
- `Views/MediaStudioView.axaml` — 🌿 分支按钮

**实施内容：**
- AI 消息操作栏新增"🌿 分支"按钮
- 点击后触发 `ForkRequested` 事件，传递源 ViewModel 和目标消息
- 提供 `GetMessagesUpTo(message)` 方法获取分支所需的消息子集
- **事件驱动解耦**：模块不关心分支 Session 的创建逻辑，由宿主应用自行处理

### 3.5 ✅ Token 用量显示

**改动文件：**
- `Models/MediaGenSession.cs` — `MediaChatMessage` 新增 `PromptTokens` / `CompletionTokens`
- `ViewModels/MediaSessionViewModel.cs` — `ChatMessageViewModel` 新增 Token 属性
- `Views/MediaStudioView.axaml` — Token 用量文本显示

**实施内容：**
- `MediaChatMessage` 模型新增 `PromptTokens` / `CompletionTokens` 可空字段
- `ChatMessageViewModel` 新增 `HasTokenUsage` 和 `TokenUsageText` 属性
- AI 消息气泡中显示 `⚡ 输入 X / 输出 X tokens`（仅有数据时可见）
- **待后续完善**：API 响应中的 `usage` 字段解析（需修改 `AiInsightService` 流式处理）

## 四、P2 体验优化

### 4.1 ✅ Mermaid 图表渲染

**改动文件：** `Controls/Markdown/MarkdownRenderer.cs`

**实施内容：**

### 4.3 ✅ 拖拽上传文件/图片

**改动文件：**
- `Views/MediaStudioView.axaml` — 聊天区域 `DragDrop.AllowDrop="True"`

**实施内容：**
- 聊天区域启用 DragDrop
