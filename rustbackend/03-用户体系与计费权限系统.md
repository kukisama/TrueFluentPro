## 1. 设计总纲

### 1.1 三个关键"可选"

| 模块 | 默认状态 | 管理员可配 | 说明 |
|------|---------|-----------|------|
| **认证** | 开启 (本地 JWT) | 可切换 AAD / 本地 | 至少一种必须开启 |
| **计费** | **关闭** | `billing.enabled = true` 开启 | 不计费 = 无限额度 |
| **配额** | 跟随计费 | 可独立开启 | 计费关时配额也关；可只开配额不开计费 |

> **核心原则**：管理员不开计费 → 系统完全不做用量统计和配额拦截，零额外开销。

### 1.2 模块依赖关系

```
┌──────────────────────────────────────────┐
│           认证模块 (必选)                  │
│  AAD JWT / 本地账密 JWT                   │
└──────────────┬───────────────────────────┘
               │ claims: user_id, role, tenant_id
               ▼
┌──────────────────────────────────────────┐
│         用户管理模块 (必选)                │
│  用户表, 角色, 能力可见性                 │
└──────────────┬───────────────────────────┘
               │
       ┌───────┴────────┐
       ▼                ▼
┌──────────────┐  ┌──────────────────────┐
│ 配额模块      │  │ 计费模块 [可选]        │
│ [可选]        │  │ 计量 + 套餐 + 账单    │
│ 内存/Redis    │  │ 依赖配额模块           │
└──────────────┘  └──────────────────────┘
```

## 2. 认证系统

### 2.1 双模认证

系统支持两种认证方式，通过配置切换：

```toml
# config/default.toml

[auth]
# "aad" — Azure AD JWT 验证 (生产推荐)
# "local" — 本地账密 + 自签 JWT (极简部署)
mode = "local"

[auth.aad]
tenant_id = ""
client_id = ""
audience = "api://truefluentpro"
issuer = "https://login.microsoftonline.com/{tenant_id}/v2.0"

[auth.local]
jwt_secret = ""          # 首次启动时自动生成
jwt_expiry_hours = 24
allow_registration = false   # 管理员控制是否允许注册
```

### 2.2 AAD 模式

```
客户端 → MSAL.NET PKCE → AAD → access_token (aud=你的后台)
                                       │
                                       ▼
网关: JWT 验签 (aud + iss + exp + 签名)
     提取 claims: oid (用户ID), tid (租户ID), roles, scp
     首次访问 → 自动创建 users 记录 (JIT provisioning)
```

**技术选型**：`jsonwebtoken 9.x` + `jwks-client-rs`（JWKS 自动缓存 5 分钟 TTL）

### 2.3 本地模式

```
客户端 → POST /v1/auth/login { username, password }
                                       │
                                       ▼
网关: argon2id 密码校验 → 签发自签 JWT (HS256)
     claims: sub (user_id), role, exp
```

```
客户端 → POST /v1/auth/register { username, password, invite_code }
     → 管理员 allow_registration=true 时才可用
     → invite_code 可选（管理员发放）
```

**技术选型**：`argon2 0.5`（密码 hash，参数：time_cost=3, memory=65536 KiB, parallelism=1）+ `jsonwebtoken 9.x`（签发/校验）

### 2.4 认证端点

```
─── 认证端点 (无需 token) ───
POST  /v1/auth/login              → 本地模式登录
POST  /v1/auth/register           → 本地模式注册 (可选)
POST  /v1/auth/refresh            → 刷新 token

─── AAD 模式下这些端点返回 501 (Not Implemented) ───
```

### 2.5 Auth Middleware 设计

```
Auth Middleware (tower Layer)
  │
  ├─ 读 config.auth.mode
  │
  ├─ mode == "aad"
  │    → 验证 Bearer token via JWKS
  │    → 提取 AadClaims { oid, tid, roles, scp }
  │    → 转换为内部 UserContext { user_id, tenant_id, role }
  │
  ├─ mode == "local"
  │    → 验证 Bearer token via HS256 secret
  │    → 提取 LocalClaims { sub, role }
  │    → 转换为内部 UserContext { user_id, tenant_id="default", role }
  │
  └─ 将 UserContext 注入 request extensions
```

> **关键**：所有下游 Handler 只依赖 `UserContext`，不关心认证是 AAD 还是本地。

## 3. 用户管理

### 3.1 数据库模型

```sql
-- ═══ 用户表 ═══
CREATE TABLE users (
    id              TEXT PRIMARY KEY,       -- AAD: oid; 本地: UUID
    username        TEXT,                   -- 本地模式的用户名
    display_name    TEXT NOT NULL,
    email           TEXT,
    password_hash   TEXT,                   -- 本地模式: argon2id hash; AAD模式: NULL
    role            TEXT NOT NULL DEFAULT 'user',  -- 'user' / 'admin'
    plan_id         TEXT REFERENCES subscription_plans(id) DEFAULT 'free',
    is_active       BOOLEAN NOT NULL DEFAULT TRUE,
    auth_provider   TEXT NOT NULL DEFAULT 'local', -- 'aad' / 'local'
    tenant_id       TEXT NOT NULL DEFAULT 'default',
    metadata_json   JSONB,                  -- 扩展信息
    first_seen_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    last_seen_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE UNIQUE INDEX idx_users_username ON users(username) WHERE username IS NOT NULL;
```

### 3.2 JIT 用户创建 (Just-In-Time Provisioning)

**AAD 模式**：用户首次携带有效 token 访问任意端点时：

1. Auth Middleware 校验 token 通过。
2. 查 `users` 表是否存在 `id = claims.oid` 的记录。
3. 不存在 → 自动插入：
   ```sql
   INSERT INTO users (id, display_name, email, role, auth_provider, tenant_id)
   VALUES ($oid, $preferred_username, $email, 'user', 'aad', $tid)
   ON CONFLICT DO NOTHING;
   ```
4. 已存在 → 更新 `last_seen_at`。

**本地模式**：用户必须先注册 (`POST /v1/auth/register`)。

### 3.3 首个管理员

- **首次启动向导** (详见文档 04) 会引导创建第一个管理员账户。
- AAD 模式：第一个登录的用户自动标记为 admin。
- 本地模式：向导中创建 admin 账户。

### 3.4 角色与权限

| 角色 | 权限 |
|------|------|
| `user` | 调用已启用的能力端点、查看自己的用量和配额 |
| `admin` | user 的所有权限 + Admin API (管理用户/Provider/凭据/计费配置) |

```
权限检查:
  ├─ 能力端点: UserContext.role ∈ ['user', 'admin']
  │            + CapabilityRegistry 该能力已启用
  │            + 配额未超 (如果计费模块开启)
  │
  └─ Admin 端点: UserContext.role == 'admin'
                  否则 → 403 Forbidden
```


### 4.2 能力状态枚举

| 状态 | 含义 |
|------|------|
| `available` | 可正常使用 |
| `quota_exceeded` | 配额已用完（计费开启时） |
| `disabled` | 管理员未启用此能力 |
| `unavailable` | 能力已启用但无可用 Provider |

### 4.3 能力列表构建逻辑

```
fn build_user_capabilities(user, registry, billing_engine):
    result = []
    for cap in registry.all_capabilities():
        if not cap.is_enabled:
            result.push({ id: cap.id, status: "disabled" })
            continue

        if not registry.has_available_providers(cap.id):
            result.push({ id: cap.id, status: "unavailable" })
            continue

        status = "available"
        quota = None

        if billing_engine.is_enabled():
            usage = billing_engine.get_usage(user.id, cap.resource_type)
            limit = billing_engine.get_limit(user.plan_id, cap.resource_type)
            if usage >= limit:
                status = "quota_exceeded"
            quota = { used: usage, limit: limit, ... }

        result.push({ id: cap.id, status, quota, ...cap.metadata })
    return result
```

## 5. 计费模块（可选耦合）

### 5.1 开关控制

```toml
# config/default.toml

[billing]
enabled = false           # 默认关闭

[billing.quota]
enabled = false           # 可独立于计费开启
check_mode = "pre"        # "pre" (调用前检查) / "post" (调用后扣减) / "both"
```

### 5.2 BillingEngine trait

```
trait BillingEngine {
    fn is_enabled() -> bool

    // 计量: 记录一次调用的用量
    async fn record_usage(RecordUsageRequest) -> Result<()>

    // 配额: 检查是否超限
    async fn check_quota(user_id, resource_type) -> Result<QuotaStatus>

    // 查询: 用户用量
    async fn get_usage(user_id, resource_type, period) -> Result<UsageSummary>

    // 管理: 套餐管理
    async fn get_plan(plan_id) -> Result<SubscriptionPlan>
    async fn update_plan(plan_id, changes) -> Result<()>
}
```

### 5.3 两种实现

```
┌─────────────────────────────────────────┐
│  DisabledBillingEngine (默认)            │
│                                         │
│  is_enabled() → false                   │
│  record_usage() → 直接 return Ok(())    │
│  check_quota() → return Ok(Unlimited)   │
│  get_usage() → return empty             │
│                                         │
│  零额外开销，零 DB 查询                  │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  ActiveBillingEngine (管理员开启后)       │
│                                         │
│  数据存储:                               │
│    usage_events 表 — 每次调用明细         │
│    monthly_usage 表 — 月度聚合缓存        │
│    subscription_plans 表 — 套餐定义       │
│                                         │
│  配额检查:                               │
│    有 Redis → INCRBY + TTL (原子操作)    │
│    无 Redis → 内存 DashMap + 异步写 DB   │
│                                         │
│  计量写入:                               │
│    异步批量写入 (不阻塞请求)              │
│    每 5 秒或满 100 条 flush 到 DB        │
└─────────────────────────────────────────┘
```

### 5.4 计费数据模型

```sql
-- ═══ 套餐定义 ═══
CREATE TABLE subscription_plans (
    id              TEXT PRIMARY KEY,       -- 'free', 'basic', 'pro', 'unlimited'
    display_name    TEXT NOT NULL,
    price_monthly   REAL,                   -- 0 = 免费; NULL = 自定义
    limits_json     JSONB NOT NULL,         -- 各能力月度上限
    is_active       BOOLEAN NOT NULL DEFAULT TRUE,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- limits_json 示例:
-- {
--   "chat_token": 500000,
--   "speech_minute": 60,
--   "image": 50,
--   "video_second": 60,
--   "translate_minute": 120,
--   "tts_character": 100000
-- }
-- 特殊值: -1 = 无限制; 0 = 不可用

-- ═══ 用量事件 (每次调用记录) ═══
CREATE TABLE usage_events (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,   -- SQLite; PG 用 BIGSERIAL
    user_id         TEXT NOT NULL REFERENCES users(id),
    capability_id   TEXT NOT NULL,          -- 'chat', 'text.translate', ...
    resource_type   TEXT NOT NULL,          -- 'chat_token', 'speech_minute', ...
    amount          BIGINT NOT NULL,        -- token 数 / 秒数 / 次数
    provider_id     TEXT,                   -- 哪个 Provider 服务的
    model           TEXT,                   -- 模型名称
    endpoint_path   TEXT,                   -- 请求路径
    upstream_status INT,                    -- 上游响应码
    latency_ms      INT,                   -- 代理延迟
    request_id      TEXT,                   -- 追踪 ID
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_usage_user_date ON usage_events(user_id, created_at);
CREATE INDEX idx_usage_cap_date ON usage_events(capability_id, created_at);

-- ═══ 月度汇总 (定期聚合，加速配额查询) ═══
CREATE TABLE monthly_usage (
    user_id         TEXT NOT NULL REFERENCES users(id),
    resource_type   TEXT NOT NULL,
    year_month      TEXT NOT NULL,          -- '2026-04'
    total_amount    BIGINT NOT NULL DEFAULT 0,
    updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (user_id, resource_type, year_month)
);
```

### 5.5 计量策略

| 能力 | 资源类型 | 计量单位 | 如何获取用量 |
|------|---------|---------|-------------|
| `chat` | `chat_token` | token 数 | Azure OpenAI 响应的 `usage.total_tokens` |
| `text.translate` | `translate_char` | 字符数 | 请求 text 的字符数 |
| `speech.live.translate` | `translate_minute` | 分钟 | WS 连接持续时长 |
| `speech.stt` | `speech_minute` | 分钟 | 音频文件时长 |
| `speech.tts` | `tts_character` | 字符数 | 请求 text 的字符数 |
| `image.generate` | `image` | 张 | 每次成功生成计 1 |
| `video.generate` | `video_second` | 秒 | 生成视频时长 |

### 5.6 配额检查 Middleware

```
Quota Middleware (tower Layer)
  │
  ├─ billing_engine.is_enabled() == false → 直接 next()
  │
  ├─ 读取 UserContext.user_id + 从 path 推导 resource_type
  │
  ├─ billing_engine.check_quota(user_id, resource_type)
  │    │
  │    ├─ QuotaStatus::Ok(remaining) → 继续
  │    │    └─ 设置 Response Header: X-Quota-Remaining, X-Quota-Limit
  │    │
  │    ├─ QuotaStatus::Exceeded → 返回 429
  │    │    {
  │    │      "error": "quota_exceeded",
  │    │      "message": "Monthly chat_token quota exceeded",
  │    │      "usage": 500000,
  │    │      "limit": 500000,
  │    │      "resets_at": "2026-05-01T00:00:00Z"
  │    │    }
  │    │
  │    └─ QuotaStatus::Unlimited → 继续 (无限额度套餐)
  │
  └─ 请求完成后 → billing_engine.record_usage(...)
```

### 5.7 计费模块开启/关闭的影响范围

| 计费关闭 (默认) | 计费开启 |
|----------------|---------|
| 无 `usage_events` 写入 | 每次调用写入 |
| 无配额检查 | Middleware 检查配额 |
| `GET /v1/user/profile` 中 quota 字段为 null | quota 字段有值 |
| Admin 套餐管理 API 返回 501 | 完整功能 |
| 零额外性能开销 | Redis/内存配额缓存开销 |

## 8. 安全要点

| 要求 | 实施方式 |
|------|---------|
| 用户 ID 从 JWT 提取 | **永远不信任**请求体/路径中的 user_id |
| 密码存储 | `argon2id`（time_cost=3, memory=65536, parallelism=1） |
| JWT 密钥 | AAD: 通过 JWKS 自动获取；本地: 启动时从 env/DB 获取 |
| 配额操作原子性 | Redis: `INCRBY` 原子操作；内存: `DashMap` + 无锁递增 |
| 计费数据完整性 | usage_events 异步批量写入，失败重试 3 次后落本地文件 |
| Admin API 保护 | 双重检查：JWT 有效 + `role == 'admin'` |
| 注册防滥用 | 本地模式：invite_code + rate limit |
