## 一、变更概述

本次变更从零搭建了 `rustbackend/` Rust 后端网关，包含：

1. **4 篇架构设计文档**（总体架构、插件化 Provider、用户体系与计费、数据层与部署）
2. **8 个 crate 的 Cargo workspace**，实现完整 API 网关
3. **5 个 Azure Provider 适配器**（Chat、Image、TTS、Translate、WebSocket 实时翻译）
4. **可选计费模块**、**凭据三级加密链**、**Docker 多阶段部署**
5. **全量 SSML / 翻译 / 实时翻译参数**，与桌面客户端配置中心 1:1 对齐

## 五、Azure Provider 适配器（5 个）

### 5.1 Azure OpenAI Chat（`AzureOpenAiChat`）

- 上游：`POST /openai/deployments/{deployment}/chat/completions`
- 解析 SSE `data: {...}` / `data: [DONE]` 格式
- 支持流式（SSE）和非流式（JSON）响应
- 凭据：`endpoint` + `api_key` + `deployment` + `api_version`

### 5.2 Azure OpenAI Image（`AzureOpenAiImage`）

- 上游：Azure OpenAI DALL-E API
- 返回 base64 JSON 图片
- 凭据：`endpoint` + `api_key` + `image_deployment`

### 5.3 Azure Speech TTS（`AzureSpeechTts`）

- 上游：`POST /cognitiveservices/v1`（REST API）
- 完整 SSML 构建（含 mstts 命名空间），支持全部客户端配置参数
- 凭据：`speech_endpoint` + `speech_key`
- **SSML 参数完整支持**（见下方 §六）

### 5.4 Azure Translator（`AzureTranslator`）

- 上游：`POST /translate?api-version=3.0`（REST v3.0）
- 自动语言检测、可配置 region 和 endpoint
- 凭据：`translator_endpoint` + `translator_key` + `translator_region`
- **高级参数**：profanity_action/profanity_marker（白名单校验）、text_type、category（URL 编码）、include_alignment、include_sentence_length

### 5.5 Azure Speech Translation（`AzureSpeechTranslation`）

- 上游：`wss://{region}.stt.speech.microsoft.com/stt/speech/universal/v2`
- 实现 Azure Speech Protocol 帧格式（speech.config + speech.context + binary audio frames）
- 二进制音频帧：uint16-BE header length prefix + WAV header + 16kHz/16bit/mono PCM
- 支持自动语言检测（`?detect=en-US,es-ES,fr-FR`）
- 语义分段（按句子边界 `.` `?` `!` 分段）
- 解析 `translation.response`：提取 `SpeechPhrase`（最终）和 `SpeechHypothesis`（中间结果）
- 凭据：`speech_key` + `speech_region`

| `language` | `<lang xml:lang="">` | 多语言声音的语言覆盖 |

| `sub_alias` | `<sub>` | 替代发音 |
| `raw_ssml` | — | 直接传入 SSML（跳过自动构建） |
| `output_format` | HTTP Header | 支持所有 Azure 格式名（audio-24khz-96kbitrate-mono-mp3 等） |

## 七、核心模块详情

### 7.1 认证（双模）

- **本地模式**：argon2id 密码哈希 + HS256 JWT（24h 有效期，可配置）
- **AAD 模式**：验证 Azure AD JWT（aud/iss/exp），JIT 自动创建用户
- 切换方式：`auth.mode = "local"` 或 `"aad"`（配置文件 / 环境变量）
- WebSocket 认证：通过 `?token=<jwt>` query param（浏览器 WebSocket 无法发送自定义 Header）

### 7.2 存储层

- **StorageBackend trait**：抽象接口，支持切换实现
- **SQLite 实现**（默认）：13 张表 + 自动迁移 + 默认套餐/能力定义
- 数据表：`system_config`、`users`、`user_passwords`、`capabilities`、`providers`、`capability_provider_bindings`、`provider_credentials`、`subscription_plans`、`usage_events`、`monthly_usage`、`audit_log`
- 默认套餐：Free / Basic / Pro / Unlimited
- 默认能力：chat / text.translate / speech.live.translate / speech.stt / speech.tts / image.generate / video.generate

### 7.3 凭据加密链（三级）

1. **环境变量**：`{PROVIDER_ID}_{KEY}` 格式（如 `AZURE_OPENAI_API_KEY`）
2. **数据库加密**：AES-256-GCM 加密存储（需配置 `master_key_base64`）
3. **Azure Key Vault**：预留接口（未来实现）

### 7.4 计费模块（可选）

- **DisabledBillingEngine**：零开销直通（默认，`billing.enabled = false`）
- **ActiveBillingEngine**：基于用户套餐的月度配额执行
  - 读取用户套餐限额 → 查当月已用量 → 返回 Ok / Exceeded / Unlimited
  - 用量记录到 `usage_events` + `monthly_usage` 汇总表
  - 启用方式：`GATEWAY_BILLING__ENABLED=true`

### 7.5 Provider 注册表

- **ProviderRegistry**：动态分发（azure_openai → chat+image, azure_speech → tts+live_translate, azure_translator → translate）
- 存储为 `Arc<RwLock<ProviderRegistry>>`，管理员修改 Provider 后自动热重载
- 支持多实例同类型 Provider（优先级 + 权重路由）

### 7.6 缓存层

- **InMemoryCache**：基于 DashMap 的进程内缓存
- 支持 TTL 过期、计数器（限流用）
- Redis 实现预留（配置 `cache.mode = "redis"`）
