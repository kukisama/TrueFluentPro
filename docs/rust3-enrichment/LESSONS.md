## 从 v2 继承的教训

- "编译通过"不等于"功能可用"。验收标准是 cargo test 全绿 + 可选的 API 自测。
- 禁止"似是而非"的交付。函数存在但逻辑为空 = 未完成。
- 功能参考源是 Spec 文档（.exchange/docs/），不是 C# 源码。
- 按垂直切片交付（一个功能端到端跑通），不按水平切片。
- 测试要验证行为契约，不要写快照式测试。
- C# 中的皇帝类在 Rust3 中已拆解为多个 crate，保持这个结构。
- Avalonia 的 Cursor 类型名与 WPF 不同，React 的 cursor CSS 也不同。
