# Azure Speech 容器化思考

## 一、Azure Speech SDK 自带的音频处理能力

### 1.1 结论：确实自带降噪和音频增强

Azure Speech SDK 内置了 **Microsoft Audio Stack (MAS)**，提供完整的本地音频处理管线，**全部在设备端执行，不需要云端处理**。

### 1.2 具体功能

| 功能 | 说明 |
|------|------|
| **噪声抑制 (Noise Suppression)** | 实时降低背景噪音，单麦克风可用但效果有限，麦克风阵列效果最佳 |
| **回声消除 (Echo Cancellation)** | 消除扬声器回馈到麦克风的声音，适用于免提/扬声器场景 |
| **自动增益控制 (AGC)** | 自动调整音频输入的音量水平 |
| **去混响 (Dereverberation)** | 减少房间回声对语音识别的影响 |
| **波束成形 (Beamforming)** | 麦克风阵列专用，定向采集特定方向的声音 |

### 1.3 使用方式 (C#)

需要安装 NuGet 包 `Microsoft.CognitiveServices.Speech.Extension.MAS`（SDK v1.33.0+）。

```csharp
// 启用全部默认增强
var audioProcessingOptions = AudioProcessingOptions.Create(
    AudioProcessingConstants.AUDIO_INPUT_PROCESSING_ENABLE_DEFAULT);
var audioInput = AudioConfig.FromDefaultMicrophoneInput(audioProcessingOptions);
var recognizer = new SpeechRecognizer(speechConfig, audioInput);
```

```csharp
// 关闭特定功能（如关闭降噪和回声消除）
int flags = AudioProcessingConstants.AUDIO_INPUT_PROCESSING_ENABLE_DEFAULT
    | AudioProcessingConstants.AUDIO_INPUT_PROCESSING_DISABLE_NOISE_SUPPRESSION
    | AudioProcessingConstants.AUDIO_INPUT_PROCESSING_DISABLE_ECHO_CANCELLATION;
var audioProcessingOptions = AudioProcessingOptions.Create(flags);
```

**关键标志位：**
- `AUDIO_INPUT_PROCESSING_ENABLE_DEFAULT` — 启用全部增强
- `AUDIO_INPUT_PROCESSING_DISABLE_NOISE_SUPPRESSION` — 关闭降噪
- `AUDIO_INPUT_PROCESSING_DISABLE_ECHO_CANCELLATION` — 关闭回声消除
- `AUDIO_INPUT_PROCESSING_DISABLE_GAIN_CONTROL` — 关闭自动增益
- `AUDIO_INPUT_PROCESSING_DISABLE_DEREVERBERATION` — 关闭去混响

> **参考文档：**
> - https://learn.microsoft.com/en-us/azure/ai-services/speech-service/audio-processing-overview
> - https://learn.microsoft.com/en-us/azure/ai-services/speech-service/audio-processing-speech-sdk

## 二、提升转录正确率的手段

### 2.1 短语列表 (Phrase List) — 最轻量的手段

- **原理**：在运行时动态告知识别器一组特定词汇/短语，提升这些词被识别到的概率
- **适用场景**：产品名、人名、地名、缩写、技术术语（最多约 500 条）
- **支持**：实时识别和快速转录（不支持批量转录）
- **可调权重**：0.0~2.0，控制短语对识别结果的影响力度

```csharp
var recognizer = new SpeechRecognizer(speechConfig, audioConfig);
var phraseList = PhraseListGrammar.FromRecognizer(recognizer);
phraseList.AddPhrase("Contoso Bank");
phraseList.AddPhrase("Kubernetes");
phraseList.AddPhrase("你好世界");
```

> **参考文档：** https://learn.microsoft.com/en-us/azure/ai-services/speech-service/improve-accuracy-phrase-list

### 2.2 自定义语音模型 (Custom Speech) — 系统性提升

- **原理**：上传领域文本语料 + 带标注的音频数据，在微软基线模型上进行微调训练
- **声学适应**：针对特定环境噪声（医院、工厂、法庭）进行训练
- **语言适应**：学习领域专用词汇和表达模式
- **评估**：通过 Word Error Rate (WER) 进行量化评估
- **管理**：在 Speech Studio 中管理训练数据、模型版本和端点

**适用场景：**
- 医疗（药品名、诊断术语）
- 法律（法条、法律用语）
- IT/技术（API 名称、错误码、命令行参数）

> **参考文档：** https://learn.microsoft.com/en-us/azure/ai-services/speech-service/custom-speech-overview

### 2.3 自定义显示格式 (Custom Display Format / Custom DPP)

- **自定义 ITN (Inverse Text Normalization)**：自定义口语到书面语的转换规则（如"四百块"→"¥400"）
- **自定义大小写/格式化规则**
- **自定义脏话过滤列表**
- **基于正则的替换规则**

> **参考文档：** https://learn.microsoft.com/en-us/azure/ai-services/speech-service/how-to-custom-speech-display-text-format

### 2.4 输出格式选择

SDK 支持 Detailed 输出格式，返回多种文本变体：

| 格式 | 说明 |
|------|------|
| Lexical | 原始识别词 |
| ITN | 逆文本规范化（数字、日期、货币自动格式化） |
| MaskedITN | ITN + 脏话遮罩 |
| Display | 最终显示文本（含标点、大写、ITN、脏话处理） |

### 2.5 综合建议

| 手段 | 难度 | 效果 | 适用阶段 |
|------|------|------|----------|
| Phrase List | ⭐ 低 | 对特定词汇立竿见影 | 快速验证/原型 |
| Custom Speech 模型 | ⭐⭐⭐ 高 | 系统性提升 | 长期生产环境 |
| Custom Display Format | ⭐⭐ 中 | 输出格式优化 | 后处理优化 |
| MAS 音频增强 | ⭐ 低 | 改善音频输入质量 | 噪声环境 |
| 组合使用 | — | 最佳效果 | 建议所有手段叠加使用 |

## 三、容器化部署方案

### 3.1 官方容器镜像

微软提供官方 Docker 镜像，从 MCR (Microsoft Container Registry) 拉取：

```bash
# STT (语音转文本)
docker pull mcr.microsoft.com/azure-cognitive-services/speechservices/speech-to-text:latest

# Custom STT (自定义语音模型)
docker pull mcr.microsoft.com/azure-cognitive-services/speechservices/custom-speech-to-text:latest

# Neural TTS (神经网络语音合成)
docker pull mcr.microsoft.com/azure-cognitive-services/speechservices/neural-text-to-speech:latest
```

### 3.2 运行容器

```bash
docker run -d -p 5000:5000 \
  -e "Eula=accept" \
  -e "Billing=https://<your_region>.api.cognitive.microsoft.com/" \
  -e "ApiKey=<your_api_key>" \
  mcr.microsoft.com/azure-cognitive-services/speechservices/speech-to-text:latest
```

**最低硬件要求（单容器）：**
- STT：1 vCPU + 4GB RAM
- TTS：1 vCPU + 2GB RAM
- 建议生产环境：4C/8G 以上

### 3.3 两种运行模式

| 模式 | 联网 (Connected) | 离线 (Disconnected) |
|------|-------------------|---------------------|
| **网络要求** | 需要持续连接 Azure 进行计费 | 完全离线运行 |
| **计费方式** | 按量付费（如 STT $1.00/hr）或承诺层级折扣 | 年度固定配额预付费 |
| **门槛** | 标准使用，直接可用 | 需要向微软申请审批 |
| **典型价格** | 按量 ~$1/hr；承诺层级 ~$0.65-0.80/hr | 120K hr/年 ~$74,100；600K hr/年 ~$285,000 |
| **适用场景** | 混合云、一般本地化需求 | 合规要求、气隙网络、高度安全环境 |

### 3.4 自定义模型在容器中的使用

- 容器中使用自定义模型需要通过 `MODELID` 参数指定模型 ID（从 Speech Studio 训练后获取）
- 容器本身就是端点，不需要像云端那样部署到 Custom Endpoint
- 如果需要多个模型，需要运行多个容器实例，每个配置不同的 model ID

```bash
docker run -d -p 5000:5000 \
  -e "Eula=accept" \
  -e "Billing=https://<region>.api.cognitive.microsoft.com/" \
  -e "ApiKey=<key>" \
  -e "MODELID=<your_custom_model_id>" \
  mcr.microsoft.com/azure-cognitive-services/speechservices/custom-speech-to-text:latest
```

### 3.5 Kubernetes 大规模部署

官方提供 Helm Chart：
```bash
helm install speech-stt microsoft/cognitive-services-speech-onpremise
```
支持自动扩缩、健康检查、资源限制等生产级特性。

## 四、客户端连接容器的方案设计

### 4.1 容器暴露的协议

容器暴露以下端点（与云端 API 兼容）：

| 协议 | 端点 | 用途 |
|------|------|------|
| WebSocket | `ws://host:5000/speech/recognition/multiaudio/websocket/v1` | SDK 实时识别（推荐） |
| HTTP REST | `http://host:5000/speech/recognition/conversation/cognitiveservices/v1` | REST API 短音频识别 |

### 4.2 方案 A：C# 客户端 → 容器

这是最直接的方案。C# Speech SDK 原生支持指向自定义 host。

```csharp
// 关键：用 FromHost 而不是 FromSubscription
var speechConfig = SpeechConfig.FromHost(new Uri("ws://your-container-host:5000"));
// 如果容器需要 key
speechConfig.SpeechRecognitionLanguage = "zh-CN";

var audioConfig = AudioConfig.FromDefaultMicrophoneInput();
var recognizer = new SpeechRecognizer(speechConfig, audioConfig);

// Phrase List 仍然可用
var phraseList = PhraseListGrammar.FromRecognizer(recognizer);
phraseList.AddPhrase("TrueFluentPro");

recognizer.Recognized += (s, e) => {
    Console.WriteLine($"识别结果: {e.Result.Text}");
};
await recognizer.StartContinuousRecognitionAsync();
```

**设计要点：**
- MAS 音频处理在客户端 SDK 侧执行，与容器无关，因此 C# 客户端仍可启用降噪
- Phrase List 由客户端 SDK 发送给容器，容器支持此功能
- 如果容器运行了自定义模型，客户端不需要额外配置 endpoint ID

### 4.3 方案 B：Java 客户端 → 容器

Java SDK 用法几乎完全对称：

```java
SpeechConfig speechConfig = SpeechConfig.fromHost(new URI("ws://your-container-host:5000"));
speechConfig.setSpeechRecognitionLanguage("zh-CN");

AudioConfig audioConfig = AudioConfig.fromDefaultMicrophoneInput();
SpeechRecognizer recognizer = new SpeechRecognizer(speechConfig, audioConfig);

PhraseListGrammar phraseList = PhraseListGrammar.fromRecognizer(recognizer);
phraseList.addPhrase("TrueFluentPro");

recognizer.recognized.addEventListener((s, e) -> {
    System.out.println("识别结果: " + e.getResult().getText());
});
recognizer.startContinuousRecognitionAsync().get();
```

**Java 方案的差异：**
- Java SDK **不包含 MAS 音频处理扩展**，降噪等功能不可用
- 如果 Java 端需要音频预处理，需要在应用层自行处理（如用第三方音频库做降噪后再输入 SDK）
- 或者考虑在容器前端加一层音频预处理服务

### 4.4 方案 C：Web 前端 (JavaScript/TypeScript) → 容器

Web 平台的 Speech SDK (npm: `microsoft-cognitiveservices-speech-sdk`) 同样支持连接容器。

```javascript
import * as SpeechSDK from 'microsoft-cognitiveservices-speech-sdk';

// 关键：用 fromHost 指向容器（SDK 会自动拼接正确的 WebSocket 路径）
const speechConfig = SpeechSDK.SpeechConfig.fromHost(
    new URL("ws://your-container-host:5000"),
    "" // 容器的 API key（如果需要）
);
speechConfig.speechRecognitionLanguage = 'zh-CN';

const audioConfig = SpeechSDK.AudioConfig.fromDefaultMicrophoneInput();
const recognizer = new SpeechSDK.SpeechRecognizer(speechConfig, audioConfig);

recognizer.recognized = (s, e) => {
    if (e.result.text) {
        console.log('识别结果:', e.result.text);
    }
};

recognizer.startContinuousRecognitionAsync();
```

**Web 方案的特殊考虑：**

| 问题 | 说明 |
|------|------|
| **CORS** | 容器默认可能不允许跨域，需要在容器前放置反向代理（如 Nginx）配置 CORS headers |
| **HTTPS/WSS** | 浏览器安全策略要求 HTTPS 页面只能连接 WSS，需要为容器配置 TLS 证书 |
| **音频采集** | 浏览器通过 `getUserMedia()` API 获取麦克风，需要 HTTPS 环境 |
| **没有 MAS** | Web SDK 不包含 MAS 音频处理，无法使用 SDK 内置降噪 |
| **Phrase List** | Web SDK 同样支持 Phrase List |

**推荐架构（Web 前端）：**

```
[浏览器] --WSS--> [Nginx 反向代理 (TLS + CORS)] --WS--> [Speech 容器 :5000]
```

### 4.5 三种客户端的对比

| 特性 | C# 客户端 | Java 客户端 | Web 前端 |
|------|-----------|-------------|----------|
| SDK 内置降噪 (MAS) | ✅ 支持 | ❌ 不支持 | ❌ 不支持 |
| Phrase List | ✅ | ✅ | ✅ |
| 连接方式 | `FromHost(ws://...)` | `fromHost(ws://...)` | `fromHost(ws://...)` |
| 额外基础设施 | 无 | 无 | 需要反向代理 (CORS/TLS) |
| 音频采集 | 系统麦克风/音频流 | 系统麦克风/音频流 | `getUserMedia()` |
| 适用场景 | 桌面应用 (Avalonia/WPF) | 后端服务/Android | 浏览器端 |

## 五、架构设计总览

### 5.1 推荐整体架构

```
                                    ┌─────────────────────────────────────────┐
                                    │           Docker / Kubernetes            │
                                    │                                         │
                                    │  ┌───────────────────────────────────┐  │
                                    │  │    Speech-to-Text Container       │  │
                                    │  │    (标准模型 or 自定义模型)        │  │
                                    │  │    Port: 5000 (WS + HTTP)         │  │
                                    │  └───────────────────────────────────┘  │
                                    │                                         │
                                    │  ┌───────────────────────────────────┐  │
                                    │  │    Nginx 反向代理 (可选)           │  │
                                    │  │    TLS 终止 + CORS + 负载均衡     │  │
                                    │  │    Port: 443                      │  │
                                    │  └───────────────────────────────────┘  │
                                    │                                         │
                                    └──────────────┬──────────────────────────┘
                                                   │
                        ┌──────────────────────────┼──────────────────────────┐
                        │                          │                          │
                ┌───────┴───────┐          ┌───────┴───────┐          ┌───────┴───────┐
                │   C# 桌面     │          │   Java 后端   │          │   Web 前端    │
                │  (Avalonia)   │          │  (Spring等)   │          │  (React等)    │
                │               │          │               │          │               │
                │ • MAS 降噪 ✅ │          │ • MAS ❌      │          │ • MAS ❌      │
                │ • Phrase List │          │ • Phrase List │          │ • Phrase List │
                │ • WS 直连     │          │ • WS 直连     │          │ • WSS 经代理  │
                └───────────────┘          └───────────────┘          └───────────────┘
```

### 5.2 对 TrueFluentPro 的建议

1. **短期**：先使用 Phrase List 为翻译场景中的高频术语做提升，零成本，即时生效
2. **中期**：部署 Speech 容器在本地/VPS，C# 客户端用 `FromHost` 连接，保持 MAS 降噪能力
3. **长期**：如有大量专业领域数据，训练 Custom Speech 模型并部署到容器中

### 5.3 容器化成本评估

| 场景 | 方案 | 大致成本 |
|------|------|----------|
| 开发测试 | Azure 云端 Free Tier | 免费 (5 hr/月) |
| 小规模生产 | 联网容器 + 按量计费 | ~$1/hr |
| 中等规模 | 联网容器 + 承诺层级 | ~$0.65-0.80/hr |
| 大规模/合规 | 离线容器 + 年度承诺 | ~$0.62/hr (12万 hr/年) 或 ~$0.48/hr (60万 hr/年) |
