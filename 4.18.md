## 二、容器部署矩阵

用户**自己决定**部署哪些容器，后端代码通过环境变量自动适配：

| 部署模式 | 容器数 | 组件 | 适用场景 |
|---------|--------|------|---------|
| **极简** | 1 | API (内嵌 SQLite) | 个人使用、体验、开发测试 |
| **标准** | 2 | API + PostgreSQL | 小团队生产环境 |
| **完整** | 3 | API + PostgreSQL + Redis | 多用户高并发生产环境 |

### 2.1 极简模式 (1 容器)

- 数据库用 **SQLite**（单文件，WAL 模式），volume 挂载持久化
- 限流、配额全走**内存** (`IMemoryCache` + `ConcurrentDictionary`)
- 适合 < 10 个用户

### 2.2 标准模式 (2 容器)

- 数据库用 **PostgreSQL** — 支持并发、事务、备份
- 限流、配额仍走内存（单实例 API 够用）
- 适合 10~100 个用户

### 2.3 完整模式 (3 容器)

- PostgreSQL + Redis
- Redis 做限流计数器（滑动窗口）、配额缓存（减少 DB 查询）、分布式锁
- API 可以横向扩展为多实例（Redis 共享状态）
- 适合 > 100 个用户

## 三、自定义端口 & 多实例不冲突

### 3.1 端口设计

所有端口通过**环境变量**配置，有默认值：

| 容器 | 环境变量 | 默认值 | 说明 |
|------|---------|--------|------|
| API | `API_PORT` | `5000` | 对外 HTTP 端口 |
| API | `API_HTTPS_PORT` | `5001` | 对外 HTTPS 端口 (可选) |
| PostgreSQL | `PG_PORT` | `5432` | PostgreSQL 监听端口 |
| Redis | `REDIS_PORT` | `6379` | Redis 监听端口 |

### 3.2 同一机器多实例部署

在同一台机器上部署多套 TrueFluentPro 后端（例如生产 + 测试），通过**不同端口 + 不同容器名前缀 + 不同数据卷**隔离：

```bash
# ======= 实例 A: 生产环境 =======
# .env.prod
INSTANCE_NAME=tfp-prod
API_PORT=5000
API_HTTPS_PORT=5001
PG_PORT=5432
REDIS_PORT=6379
DATA_DIR=./data/prod

# ======= 实例 B: 测试环境 =======
# .env.test
INSTANCE_NAME=tfp-test
API_PORT=5100
API_HTTPS_PORT=5101
PG_PORT=5532
REDIS_PORT=6479
DATA_DIR=./data/test
```

```bash
# 启动实例 A
docker compose --env-file .env.prod -p tfp-prod up -d

# 启动实例 B (同一台机器, 端口不冲突)
docker compose --env-file .env.test -p tfp-test up -d
```

### 3.3 容器命名规则

使用 `INSTANCE_NAME` 前缀防止容器名冲突：

```
tfp-prod-api        tfp-test-api
tfp-prod-postgres   tfp-test-postgres
tfp-prod-redis      tfp-test-redis
```

## 四、Docker Compose 定义

### 4.1 docker-compose.yml（完整版，用 profiles 控制可选组件）

```yaml
# docker-compose.yml
# 用法:
#   极简模式 (仅 API + SQLite):   docker compose up api
#   标准模式 (API + PostgreSQL):   docker compose --profile db up
#   完整模式 (API + PG + Redis):   docker compose --profile db --profile cache up

services:
  api:
    image: truefluentpro/api:latest
    container_name: ${INSTANCE_NAME:-tfp}-api
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "${API_PORT:-5000}:8080"
      - "${API_HTTPS_PORT:-5001}:8443"
    environment:
      # ─── 数据库 ───
      # 不设 DATABASE_URL 时自动回退到 SQLite
      - DATABASE_URL=${DATABASE_URL:-}
      # 如果用同 compose 的 PostgreSQL:
      # DATABASE_URL=Host=${INSTANCE_NAME:-tfp}-postgres;Port=5432;Database=truefluentpro;Username=tfp;Password=${PG_PASSWORD}
      
      # ─── Redis (可选) ───
      - REDIS_URL=${REDIS_URL:-}
      # 如果用同 compose 的 Redis:
      # REDIS_URL=${INSTANCE_NAME:-tfp}-redis:6379
      
      # ─── Azure 凭据 ───
      - AOAI_ENDPOINT=${AOAI_ENDPOINT}
      - AOAI_KEY=${AOAI_KEY:-}
      - SPEECH_KEY=${SPEECH_KEY:-}
      - SPEECH_REGION=${SPEECH_REGION:-}
      - BLOB_CONNECTION_STRING=${BLOB_CONNECTION_STRING:-}
      
      # ─── 凭据加密 ───
      - ENCRYPTION_KEY=${ENCRYPTION_KEY}
      
      # ─── AAD 验证 ───
      - AAD_TENANT_ID=${AAD_TENANT_ID}
      - AAD_CLIENT_ID=${AAD_CLIENT_ID}
      - AAD_AUDIENCE=${AAD_AUDIENCE}
      
      # ─── TLS ───
      - TLS_CERT_PATH=/certs/cert.pem
      - TLS_KEY_PATH=/certs/key.pem
      
      # ─── 其他 ───
      - ASPNETCORE_ENVIRONMENT=${ASPNETCORE_ENVIRONMENT:-Production}
    volumes:
      - ${DATA_DIR:-./data}/sqlite:/app/data          # SQLite 数据 (极简模式)
      - ${DATA_DIR:-./data}/certs:/certs:ro            # TLS 证书
      - ${DATA_DIR:-./data}/logs:/app/logs             # 日志
    depends_on:
      postgres:
        condition: service_healthy
        required: false
      redis:
        condition: service_healthy
        required: false
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
      interval: 30s
      timeout: 5s
      retries: 3

  postgres:
    image: postgres:17-alpine
    container_name: ${INSTANCE_NAME:-tfp}-postgres
    profiles: ["db"]
    ports:
      - "${PG_PORT:-5432}:5432"
    environment:
      - POSTGRES_DB=truefluentpro
      - POSTGRES_USER=tfp
      - POSTGRES_PASSWORD=${PG_PASSWORD}
    volumes:
      - ${DATA_DIR:-./data}/postgres:/var/lib/postgresql/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U tfp -d truefluentpro"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: ${INSTANCE_NAME:-tfp}-redis
    profiles: ["cache"]
    ports:
      - "${REDIS_PORT:-6379}:6379"
    command: >
      redis-server
      --maxmemory 128mb
      --maxmemory-policy allkeys-lru
      --requirepass ${REDIS_PASSWORD:-}
    volumes:
      - ${DATA_DIR:-./data}/redis:/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 3
```

## 五、API 后端代码的适配设计

### 5.1 数据库自动适配 (PostgreSQL / SQLite)

```
启动时检测 DATABASE_URL 环境变量:
  ├── 有值 → 使用 Npgsql (PostgreSQL)
  └── 空/不存在 → 使用 SQLite (文件: /app/data/truefluentpro.db)
```

代码层抽象：

```csharp
// Program.cs / Startup
var databaseUrl = Environment.GetEnvironmentVariable("DATABASE_URL");

if (!string.IsNullOrEmpty(databaseUrl))
{
    // PostgreSQL
    builder.Services.AddDbContext<AppDbContext>(o => 
        o.UseNpgsql(databaseUrl));
}
else
{
    // SQLite 回退
    builder.Services.AddDbContext<AppDbContext>(o => 
        o.UseSqlite("Data Source=/app/data/truefluentpro.db"));
}
```

#### 数据库 Schema

#### SQLite 兼容说明

当使用 SQLite 时，Schema 需要做以下调整（代码层自动处理）：

| PostgreSQL | SQLite 等价 |
|-----------|-------------|
| `BIGSERIAL` | `INTEGER PRIMARY KEY AUTOINCREMENT` |
| `TIMESTAMPTZ` | `TEXT` (ISO 8601 字符串) |
| `JSONB` | `TEXT` (JSON 字符串) |
| `BYTEA` | `BLOB` |
| `DECIMAL(10,2)` | `REAL` |
| `NOW()` | `datetime('now')` |

使用 EF Core 或 Dapper + 条件 SQL 来处理差异。推荐 EF Core（自动处理方言差异）。

### 5.2 Redis 可选适配

```
启动时检测 REDIS_URL 环境变量:
  ├── 有值 → 注册 RedisQuotaCache, RedisRateLimiter
  └── 空/不存在 → 注册 InMemoryQuotaCache, InMemoryRateLimiter
```

接口抽象：

```csharp
// ─── 配额缓存接口 ───
public interface IQuotaCache
{
    Task<long> IncrementUsageAsync(string userId, string resourceType, long amount);
    Task<long> GetCurrentUsageAsync(string userId, string resourceType);
    Task ResetUsageAsync(string userId, string resourceType);
}

// ─── 实现 1: Redis ───
public class RedisQuotaCache : IQuotaCache
{
    // 使用 Redis INCRBY, key = "usage:{userId}:{resourceType}:{yearMonth}"
    // TTL = 当月剩余天数 + 1 天
    // 优势: 原子操作, 多 API 实例共享
}

// ─── 实现 2: 内存 + DB ───
public class InMemoryQuotaCache : IQuotaCache
{
    // ConcurrentDictionary 做内存计数
    // 每次 increment 同时异步写 DB (fire-and-forget 或 batched)
    // 启动时从 monthly_usage 表加载当月数据
    // 单实例足够; 重启后从 DB 恢复
}

// ─── DI 注册 ───
var redisUrl = Environment.GetEnvironmentVariable("REDIS_URL");
if (!string.IsNullOrEmpty(redisUrl))
{
    builder.Services.AddSingleton<IConnectionMultiplexer>(
        ConnectionMultiplexer.Connect(redisUrl));
    builder.Services.AddSingleton<IQuotaCache, RedisQuotaCache>();
    builder.Services.AddSingleton<IRateLimiter, RedisRateLimiter>();
}
else
{
    builder.Services.AddSingleton<IQuotaCache, InMemoryQuotaCache>();
    builder.Services.AddSingleton<IRateLimiter, InMemoryRateLimiter>();
}
```

### 5.3 凭据管理 (不依赖 Key Vault / Managed Identity)

三级凭据查找链，按优先级：

```
1. 环境变量 (最高优先级)
   AOAI_KEY, SPEECH_KEY, BLOB_CONNECTION_STRING ...
   ↓ 如果为空
2. 数据库 credentials 表 (AES-256-GCM 加密)
   加密主密钥来源: ENCRYPTION_KEY 环境变量
   ↓ 如果也为空
3. Azure Key Vault (如果配置了)
   AKV_URL 环境变量指向 Key Vault
   通过 DefaultAzureCredential (支持 MI、SP、CLI 等)
```

```csharp
public class CredentialProvider
{
    public async Task<string?> GetCredentialAsync(string key)
    {
        // 1. 环境变量
        var envValue = Environment.GetEnvironmentVariable(key.ToUpperInvariant());
        if (!string.IsNullOrEmpty(envValue)) return envValue;

        // 2. 数据库
        var dbValue = await _credentialRepository.GetDecryptedAsync(key);
        if (!string.IsNullOrEmpty(dbValue)) return dbValue;

        // 3. Key Vault (可选)
        if (_keyVaultClient != null)
        {
            var secret = await _keyVaultClient.GetSecretAsync(key);
            return secret?.Value;
        }

        return null;
    }
}
```

### 6.2 请求处理管线

```
客户端请求
  │
  ▼
┌─────────────────────────────────────────────────────┐
│  ASP.NET Core Middleware Pipeline                    │
│                                                     │
│  ① Exception Handler                               │
│     → 全局异常捕获, 返回标准 ProblemDetails         │
│                                                     │
│  ② JWT Authentication                              │
│     → AddAuthentication().AddJwtBearer()            │
│     → 验证 AAD 签发的 token                         │
│     → 提取 oid (用户ID), roles, tid (租户ID)        │
│                                                     │
│  ③ Rate Limiter                                    │
│     → ASP.NET Core 内置 AddRateLimiter()            │
│     → 按用户 ID 限流                                │
│     → 后端: Redis(SLIDINGWINDOW) 或 内存            │
│                                                     │
│  ④ Quota Check Middleware                          │
│     → 读取用户套餐 → 查月度用量 → 判断是否超限      │
│     → 超限返回 429 + 剩余配额信息                   │
│                                                     │
│  ⑤ Endpoint Handler                               │
│     → 注入凭据 (CredentialProvider)                 │
│     → 构建上游请求 (HttpClient / WebSocket)         │
│     → 流式转发响应                                  │
│     → 计量用量 (解析 usage 字段 or 计算时长)        │
│     → 异步写入 usage_events                         │
│                                                     │
│  ⑥ Response                                       │
│     → 标准响应 + X-RateLimit-* headers              │
└─────────────────────────────────────────────────────┘
```

### 6.3 代理转发核心逻辑

```csharp
// 以 Chat Completions 为例
app.MapPost("/api/v1/chat/completions", async (
    HttpContext ctx,
    ICredentialProvider credentials,
    IQuotaCache quotaCache,
    IHttpClientFactory httpFactory) =>
{
    var userId = ctx.User.FindFirst("oid")!.Value;

    // 1. 读取请求体
    var requestBody = await ctx.Request.ReadFromJsonAsync<JsonElement>();

    // 2. 获取上游凭据
    var endpoint = await credentials.GetCredentialAsync("AOAI_ENDPOINT");
    var apiKey = await credentials.GetCredentialAsync("AOAI_KEY");

    // 3. 构建上游请求
    var client = httpFactory.CreateClient("AzureOpenAI");
    var upstreamUrl = $"{endpoint}/openai/deployments/{model}/chat/completions?api-version=2024-12-01-preview";
    
    var request = new HttpRequestMessage(HttpMethod.Post, upstreamUrl);
    request.Headers.Add("api-key", apiKey);
    request.Content = JsonContent.Create(requestBody);

    // 4. 转发并流式返回
    var response = await client.SendAsync(request, HttpCompletionOption.ResponseHeadersRead);
    
    ctx.Response.StatusCode = (int)response.StatusCode;
    ctx.Response.ContentType = response.Content.Headers.ContentType?.ToString();
    
    await response.Content.CopyToAsync(ctx.Response.Body);

    // 5. 异步计量 (从响应的 usage 字段提取 token 数)
    _ = RecordUsageAsync(userId, "chat_token", tokenCount, quotaCache);
});
```

### 7.3 ASP.NET Core 中使用证书

```csharp
// Program.cs
builder.WebHost.ConfigureKestrel(options =>
{
    var certPath = Environment.GetEnvironmentVariable("TLS_CERT_PATH");
    var keyPath = Environment.GetEnvironmentVariable("TLS_KEY_PATH");

    if (!string.IsNullOrEmpty(certPath) && File.Exists(certPath))
    {
        var cert = X509Certificate2.CreateFromPemFile(certPath, keyPath);
        options.ListenAnyIP(8443, listenOptions =>
        {
            listenOptions.UseHttps(cert);
        });
    }

    // HTTP 端口始终开放 (健康检查 / 内部通信)
    options.ListenAnyIP(8080);
});
```

### 9.3 配额检查中间件

```csharp
public class QuotaCheckMiddleware
{
    public async Task InvokeAsync(HttpContext context, IQuotaCache cache)
    {
        // 跳过不需要配额检查的端点
        if (IsExemptEndpoint(context.Request.Path)) 
        {
            await _next(context);
            return;
        }

        var userId = context.User.FindFirst("oid")?.Value;
        var resourceType = MapEndpointToResource(context.Request.Path);
        
        // 获取当月用量和上限
        var currentUsage = await cache.GetCurrentUsageAsync(userId, resourceType);
        var limit = await GetUserLimit(userId, resourceType);

        if (currentUsage >= limit)
        {
            context.Response.StatusCode = 429;
            await context.Response.WriteAsJsonAsync(new 
            {
                error = "quota_exceeded",
                message = $"Monthly {resourceType} quota exceeded",
                usage = currentUsage,
                limit = limit,
                resets_at = GetNextMonthStart()
            });
            return;
        }

        // 添加配额信息到响应头
        context.Response.OnStarting(() =>
        {
            context.Response.Headers["X-Quota-Remaining"] = 
                (limit - currentUsage).ToString();
            context.Response.Headers["X-Quota-Limit"] = limit.ToString();
            return Task.CompletedTask;
        });

        await _next(context);
    }
}
```

## 十、WebSocket 实时翻译桥接

```
Client ←WSS→ ASP.NET Core ←WSS→ Azure Speech

具体流程:
1. 客户端通过 WSS 连接 /wss/v1/translate?from=zh-CN&to=en
2. 后端验证 JWT, 检查配额
3. 后端用自己的 Speech Key 建立到 Azure Speech 的 WSS 连接
4. 双向转发:
   Client→Backend: 音频帧 (binary)
   Backend→Azure: 音频帧 (binary) 
   Azure→Backend: 识别/翻译结果 (JSON)
   Backend→Client: 识别/翻译结果 (JSON)
5. 任一端断开时, 清理另一端连接
6. 计量: 记录连接持续时长 (秒)
```

```csharp
app.MapGet("/wss/v1/translate", async (HttpContext context) =>
{
    if (!context.WebSockets.IsWebSocketRequest)
    {
        context.Response.StatusCode = 400;
        return;
    }

    var userId = context.User.FindFirst("oid")!.Value;
    var clientWs = await context.WebSockets.AcceptWebSocketAsync();
    
    // 建立到 Azure Speech 的 WebSocket
    var speechKey = await _credentials.GetCredentialAsync("SPEECH_KEY");
    var speechRegion = await _credentials.GetCredentialAsync("SPEECH_REGION");
    var azureWs = new ClientWebSocket();
    // ... 配置 Azure Speech WebSocket URL 和 headers ...
    await azureWs.ConnectAsync(azureUri, CancellationToken.None);

    var startTime = DateTime.UtcNow;

    // 双向桥接
    var clientToAzure = PipeAsync(clientWs, azureWs);
    var azureToClient = PipeAsync(azureWs, clientWs);
    
    await Task.WhenAny(clientToAzure, azureToClient);

    // 清理 & 计量
    var duration = DateTime.UtcNow - startTime;
    await RecordUsageAsync(userId, "translate_minute", 
        (long)Math.Ceiling(duration.TotalMinutes));
});
```
