## 二、待完成项（第二阶段 TODO）

### TODO-1：子 ViewModel DI 注入（优先级：高）

**现状**：6 个子 VM 全部在 `MainWindowViewModel` 构造函数中手动 `new` 创建，未注册到 DI 容器。
**计划要求**（见 计划3.14.md §4.3）：所有子 VM 注册为 Singleton 并通过构造函数注入。

待办：
- [ ] 在 `App.axaml.cs` 的 `ServiceCollection` 中注册全部 6 个子 ViewModel
- [ ] `MainWindowViewModel` 构造函数改为接收子 VM 注入参数，移除 `new` 调用
- [ ] 消除子 VM 构造中的大量 lambda 回调（单个 VM 最多传了 8 个 lambda），改为接口或 Messenger

### TODO-2：接口抽象层（优先级：高）

**现状**：计划中的 4 个接口全部缺失，服务层仍为直接调用/静态单例。
**计划要求**（见 计划3.14.md §4.1 第 2、3 刀）：建立跨平台抽象接口。

待办：
- [ ] `IAudioPlayer` — 抽象 NAudio `WaveOutEvent`/`AudioFileReader`，注册 `NaudioAudioPlayer` 实现
- [ ] `IAudioDeviceEnumerator` — 抽象 `AudioDeviceEnumerator` 静态方法，注册 `WasapiAudioDeviceEnumerator` 实现
- [ ] `IPathManager` — 抽象 `PathManager.Instance`，注册现有实现为 DI Singleton
- [ ] `IStatusReporter` — 抽象 `StatusMessage` 的上报方式，替代当前的 `Action<string>` lambda

### TODO-3：WeakReferenceMessenger 通信（优先级：中）

**现状**：子 VM 间通信全靠构造函数 lambda 回调和事件订阅，主 VM 手动转发配置变更。

## 四、第一阶段小结

**架构改善**：
- 从单文件 5,201 行皇帝类 → 6 个独立子 ViewModel + 3 个独立 View
- 从 RadioButton 模式切换 → FluentAvalonia NavigationView 侧栏导航
- 从手动 `new MainWindowViewModel()` → DI 容器创建主 VM

**遗留债务**：
- 子 VM 虽然文件独立，但通过大量 lambda 紧耦合到主 VM 构造函数（"拆了文件，没解耦依赖"）
- 缺少接口抽象层，跨平台化铺垫尚未建立
- 缺少 Messenger 通信，配置变更仍需手动逐个通知

**第二阶段方向**：完成 TODO-1 ~ TODO-4，实现真正的 DI 解耦 + Messenger 通信 + 接口抽象，使子 VM 可独立测试、可替换实现。
