## 3. Phase 1：DI 容器引入

### 3.0 目标

引入 `Microsoft.Extensions.DependencyInjection` + `CommunityToolkit.Mvvm`，将 ViewModel 改为由 DI 创建，为后续拆分铺路。**采用渐进策略：第一步只注册最核心的服务，其余保持 `new()`。**

### 3.1 步骤

#### Step 1-1：安装 NuGet 包

```bash
dotnet add package Microsoft.Extensions.DependencyInjection
dotnet add package CommunityToolkit.Mvvm
```

**验证点：** `dotnet build` 零 error。

#### Step 1-2：升级 ViewModelBase

```csharp
// 之前：手写 INotifyPropertyChanged
public class ViewModelBase : INotifyPropertyChanged { ... }

// 之后：继承 CommunityToolkit.Mvvm
using CommunityToolkit.Mvvm.ComponentModel;

public class ViewModelBase : ObservableObject { }
```

**注意：** `ObservableObject` 已内建 `SetProperty()` 和 `OnPropertyChanged()`，签名兼容，现有调用无需改动。

**验证点：** `dotnet build` 零 error（如果有签名不兼容的地方，逐个修复）。

#### Step 1-3：在 App.axaml.cs 建立 DI 容器

```csharp
using Microsoft.Extensions.DependencyInjection;

public partial class App : Application
{
    public static IServiceProvider Services { get; private set; } = null!;

    public override void OnFrameworkInitializationCompleted()
    {
        CrashLogger.HookAvaloniaUiThread();

        var services = new ServiceCollection();
        
        // --- 基础服务（Phase 1 只注册这些） ---
        services.AddSingleton<ConfigurationService>();
        services.AddSingleton<AzureSubscriptionValidator>();
        
        // --- ViewModel ---
        services.AddSingleton<MainWindowViewModel>();

        Services = services.BuildServiceProvider();

        if (ApplicationLifetime is IClassicDesktopStyleApplicationLifetime desktop)
        {
            desktop.ShutdownMode = ShutdownMode.OnMainWindowClose;
            var mainWindow = new MainWindow();
            var vm = Services.GetRequiredService<MainWindowViewModel>();
            vm.SetMainWindow(mainWindow);
            mainWindow.DataContext = vm;
            desktop.MainWindow = mainWindow;
        }

        base.OnFrameworkInitializationCompleted();
    }
}
```

#### Step 1-4：改造 MainWindowViewModel 构造函数接收 DI 注入

```csharp
// 之前（.cs 文件构造函数内部）：
_configService = new ConfigurationService();
_subscriptionValidator = new AzureSubscriptionValidator();

// 之后：
public MainWindowViewModel(
    ConfigurationService configService,
    AzureSubscriptionValidator subscriptionValidator)
{
    _configService = configService;
    _subscriptionValidator = subscriptionValidator;
    // ... 其余逻辑保持不变，其他服务仍 new()
}
```

**Phase 1 只改这两个注入参数，其余 `new AiInsightService()` 等保持原样。** 这是为了最小化改动范围，确保 DI 机制本身可靠。

#### Step 1-5：修改 MainWindow.axaml.cs

```csharp
// 之前：
_viewModel = new MainWindowViewModel();

// 之后：删除这行，ViewModel 已由 App.axaml.cs 设置 DataContext
// 仅保留 SetMainWindow / NotifyMainWindowShown 调用
```

### 4.2 拆分时的共享状态方案

各子 ViewModel 之间需要共享 `AzureSpeechConfig` 和一些状态标记。使用 `CommunityToolkit.Mvvm` 的 `WeakReferenceMessenger` 替代手动事件：

```csharp
// 定义消息类型
public record ConfigChangedMessage(AzureSpeechConfig Config);
public record TranslationStateMessage(bool IsTranslating);
public record StatusMessage(string Text);
public record NavigateToTimeMessage(TimeSpan Time);

// 发送方（例如 ConfigViewModel 配置修改后）：
WeakReferenceMessenger.Default.Send(new ConfigChangedMessage(newConfig));

// 接收方（例如 AiInsightViewModel）：
WeakReferenceMessenger.Default.Register<ConfigChangedMessage>(this, (r, m) =>
{
    _config = m.Config;
    OnPropertyChanged(nameof(IsAiConfigured));
});
```

**优势：** 自动弱引用，无内存泄漏风险；发送方不知道接收方是谁，解耦彻底。
