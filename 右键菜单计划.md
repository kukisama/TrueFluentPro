# 右键菜单计划

> 编制日期：2026-03-04  
> 依赖前置文档：`右键菜单在avaloniaui和FluentAvalonia中的特殊定义.md`、`Crash日志记录.md`

---

## 一、现状总结

### 1.1 三类右键菜单场景

项目中存在三种不同上下文的右键菜单使用场景，行为差异巨大：

| 场景 | 宿主控件 | 视觉树层级 | 方案 | 状态 |
|------|---------|-----------|------|------|
| **A. 音频文件列表**（ReviewModeView） | `AudioFileListBox` | NavigationView → SplitView → Panel → ReviewModeView 深处 | ~~`InlineListContextMenuController`~~ → **`MenuFlyout.ShowAt()`** | ✅ 已迁移 |
| **B. 字幕列表**（ReviewModeView） | `SubtitleCueListBox` | 同 A，NavigationView → SplitView 深处 | **`MenuFlyout.ShowAt()`**（跳转并播放 / 复制字幕文本） | ✅ 已实现 |
| **C. Media Studio 会话列表** | `SessionListBox` | 嵌入 NavigationView（`MediaStudioView`） | ~~XAML 声明式 ContextMenu~~ → **`MenuFlyout.ShowAt()`** | ✅ 已迁移 |
| **D. Media Studio 聊天消息** | `Border`（user-msg / ai-msg） | 同 C，NavigationView 深处 | **`MenuFlyout.ShowAt()`**（复制 / 全选 / 删除） | ✅ 已实现 |

### 1.2 核心问题回顾

根据《右键菜单在avaloniaui和FluentAvalonia中的特殊定义.md》的 10 组实验，**原生 `ContextMenu`（Popup 机制）在 FluentAvalonia `NavigationView` → `SplitView.Content` 内部完全不渲染**：

- 实验 8（最纯粹原生方案）：`ContextRequested` 确实触发且选中成功，但菜单 Popup 不可见
- 实验 6/7：`ContextMenu.Open()` 返回 `isOpen=True` 但 Popup 不渲染
- 实验 10：即使 `PlacementTarget = TopLevel` 也无效

这不是代码写法问题，而是 **Popup 层在 SplitView 内部的渲染机制限制**。

### 1.3 已验证可用方案（共 3 种 + 1 种失败）

经过实验 A–F 逐一验证，在 FluentAvalonia `NavigationView → SplitView.Content` 内部，**共有 3 种方式可以正常显示右键菜单**，1 种确认不可用：

#### ❌ 不可用：XAML 声明式 `ContextMenu`

原生 `ContextMenu` 在 NavigationView 内部 **Popup 完全不渲染**（实验 A 确认）。`ContextMenu.Open()` 返回 `isOpen=True` 但视觉上不可见。这是 Popup 层在 SplitView 内部的渲染机制限制，非代码写法问题。

#### ✅ 方法一：SelectableTextBlock 内置 ContextMenu

`SelectableTextBlock` 自带的 Copy / Select All 右键菜单可以正常渲染。推测它使用了特殊的 Popup 机制或不同的弹出路径。**但此菜单无法自定义菜单项**，只能提供复制和全选两个内置功能。

- **限制**：无法添加自定义菜单项（如"删除此条记录"）
- **适用场景**：纯文本选择复制需求（无需额外操作项时）

#### ✅ 方法二：手动 `new Popup()`（实验 C 验证）

完全绕开 `ContextMenu` 的 Popup 管理，手工创建 Popup 并挂到 Window 层：

```csharp
var popup = new Popup
{
    PlacementTarget = TopLevel.GetTopLevel(this) as Control,
    Placement = PlacementMode.Pointer,
    IsLightDismissEnabled = true,
    Child = new Border
    {
        Background = Brushes.White,
        Child = new TextBlock { Text = "测试菜单" }
    }
};
((TopLevel.GetTopLevel(this) as Window)!.Content as Panel)!.Children.Add(popup);
popup.IsOpen = true;
```

- **原理**：将 Popup 添加到 Window 的内容面板中，绕过 SplitView 的 Popup 层级限制
- **缺点**：需要手动管理 Popup 生命周期、样式、动画、LightDismiss；无主题跟随
- **适用场景**：需要完全自定义弹出内容的高级场景

#### ✅ 方法三：`MenuFlyout.ShowAt()`（实验 D 验证）⭐ 最终选用

Avalonia 原生的 `MenuFlyout`，通过 `FlyoutBase.ShowAt()` 使用**不同于 ContextMenu 的 Overlay 弹出路径**：

```csharp
var flyout = new MenuFlyout();
flyout.Items.Add(new MenuItem { Header = "菜单项" });
flyout.ShowAt(targetControl, showAtPointer: true);
```

- **原理**：`FlyoutBase.ShowAt()` 走 Avalonia 的 Overlay 层（`OverlayPopupHost`），而非 ContextMenu 使用的标准 `PopupRoot` 路径
- **优点**：自动跟随 FluentAvalonia 主题、支持 `MenuItem` / `Separator`、自带 LightDismiss、API 简洁
- **缺点**：每次右键需要在代码中 new 实例（不如 XAML 声明式简洁）
- **适用场景**：所有 NavigationView 内部的右键菜单需求

> **最终选择**：全部场景统一采用 **方法三 `MenuFlyout.ShowAt()`**，它在正确性、可维护性、主题一致性上是最优解。

#### 已废弃：`InlineListContextMenuController<TItem>`

此前的 workaround 方案，在 `ListBox` 同级 `Grid` 中放置非 Popup 的 `Border`，通过 `TranslateTransform` + `IsVisible` 模拟右键菜单。**现已被 MenuFlyout 方案完全替代**，控件文件 `InlineListContextMenuController.cs` 保留但不再被任何代码引用。

// 在 controller 中新增
public void HandleContextRequested(ContextRequestedEventArgs e)
{
    var selected = _selectedItemProvider();
    if (selected == null) return;

    CurrentItem = selected;
    // 键盘触发时使用选中项中心位置
    _lastRightPressPosition = EstimateSelectedItemCenter();
    _lastRightReleasePosition = _lastRightPressPosition;

    if (TryShowInternal(selected, out var shownAt))
    {
        e.Handled = true;
        MenuShown?.Invoke(selected, shownAt);
    }
}
```

## 五、实验执行结果（2026-03-05）

> 以下记录各实验的实际执行结果和关键发现。

### 实验 A 结果：❌ ContextMenu 不渲染

在嵌入模式 `MediaStudioView` 中，对 `SessionListBox` 和消息 `Border` 的原生 XAML `ContextMenu` 右键测试：
- `ContextMenu` 事件触发正常，`isOpen=True`
- **Popup 完全不可见** — 确认与 ReviewModeView 相同的问题
- 结论：NavigationView → SplitView.Content 内部**所有标准 ContextMenu Popup 均不渲染**

### 实验 B 结果：视觉树 ClipToBounds 诊断

通过 `AttachedToVisualTree` 遍历视觉树并输出 `ClipToBounds`，发现 **4 层 ClipToBounds=True**：

```
[0]  MediaStudioView    ClipToBounds=True
[4]  Border             ClipToBounds=True
[8]  SplitView          ClipToBounds=True
[11] NavigationView     ClipToBounds=True

## 六、技术实现详情（2026-03-05 落地）

### 6.1 统一 MenuFlyout 方案的事件绑定模式

所有场景采用统一模式：在 `PointerReleased` 中创建并展示 `MenuFlyout`。

**关键代码模式**：
```csharp
private void SomeControl_PointerReleased(object? sender, PointerReleasedEventArgs e)
{
    if (e.InitialPressMouseButton != MouseButton.Right) return;
    if (sender is not Control control) return;
    // ... 获取数据上下文 ...

    var flyout = new MenuFlyout();
    flyout.Items.Add(new MenuItem { Header = "菜单项" });
    flyout.ShowAt(control, showAtPointer: true);
    e.Handled = true;
}
```

**XAML 侧**：直接在控件上声明 `PointerReleased` 事件，或通过 `AddHandler` 在隧道阶段注册。

**对于 ListBox 场景**（需要精确定位到哪个 Item 被右键）：
```csharp
// 通用范型方法：从点击源向上遍历视觉树查找绑定项
private static T? ResolveHitItem<T>(object? source) where T : class
{
    var v = source as Visual;
    while (v != null)
    {
        if (v.DataContext is T item) return item;
        v = v.GetVisualParent() as Visual;
    }
    return null;
}
```

### 6.2 SelectableTextBlock 选区保留技术

**问题**：右键点击 `SelectableTextBlock` 时，控件在 `PointerPressed` 阶段清除已有选区。当 `PointerReleased` 触发我们的 MenuFlyout 时，`SelectedText` 已为空。

**解决方案**：在**隧道阶段** `PointerPressed` 中提前捕获选中文本。

```
事件时序：
  PointerPressed (Tunnel) ← 我们先拦截，保存 SelectedText
       ↓
  PointerPressed (Bubble) ← SelectableTextBlock 内部处理，清除选区
       ↓
  PointerReleased (Bubble) ← 我们的 MenuFlyout 使用已捕获的文本
```

**实现**：

```csharp
// 字段
private string? _capturedSelection;

// 在 Initialize() 中注册隧道阶段处理器
this.AddHandler(
    InputElement.PointerPressedEvent,
    CaptureSelectionBeforeRightClick,
    Avalonia.Interactivity.RoutingStrategies.Tunnel);

// 隧道阶段：在 SelectableTextBlock 处理 PointerPressed 之前保存选区
private void CaptureSelectionBeforeRightClick(object? sender, PointerPressedEventArgs e)
{
    if (!e.GetCurrentPoint(this).Properties.IsRightButtonPressed) return;
    _capturedSelection = null;

    var hit = this.InputHitTest(e.GetPosition(this)) as Visual;
    for (var v = hit; v != null; v = v.GetVisualParent() as Visual)
    {
        if (v is SelectableTextBlock stb)
        {
            _capturedSelection = stb.SelectedText;
            return;
        }
    }
}
```

**在 MenuFlyout 中使用**：
```csharp
var captured = _capturedSelection;
_capturedSelection = null;

var copyItem = new MenuItem { Header = "复制" };
copyItem.Click += async (_, _) =>
{
    var text = !string.IsNullOrEmpty(captured) ? captured : msg.Text;
    if (!string.IsNullOrEmpty(text))
        await TopLevel.GetTopLevel(this)!.Clipboard!.SetTextAsync(text);
};
```

### 6.3 禁用 SelectableTextBlock 内置菜单

当使用统一 MenuFlyout 时，需要禁用 `SelectableTextBlock` 自带的 Copy / Select All 菜单，避免两套菜单冲突：

```xml
<SelectableTextBlock Text="{Binding Text}"
                     TextWrapping="Wrap"
                     MaxWidth="500"
                     ContextFlyout="{x:Null}"/>
```

设置 `ContextFlyout="{x:Null}"` 即可阻止内置菜单弹出，右键事件正常冒泡到外层 `Border` 的 `PointerReleased` 处理器。

## 十一、结论（已更新 2026-03-05）

### 核心发现

1. **原生 `ContextMenu` 在 NavigationView 内不可用** — 这是 Popup 层在 SplitView 内渲染机制的限制，非代码问题
2. **3 种可用替代方案已验证**：SelectableTextBlock 内置菜单、手动 Popup、MenuFlyout.ShowAt()
3. **`MenuFlyout.ShowAt()` 是最优解** — 走 `OverlayPopupHost` 路径绕过限制，自带主题跟随和 LightDismiss
