> 编制日期：2026-03-04  
> 依赖前置文档：`右键菜单在avaloniaui和FluentAvalonia中的特殊定义.md`、`Crash日志记录.md`

---

## 一、现状总结

### 1.1 三类右键菜单场景

项目中存在三种不同上下文的右键菜单使用场景，行为差异巨大：

| 场景 | 宿主控件 | 视觉树层级 | 方案 | 状态 |
|------|---------|-----------|------|------|
| **A. 音频文件列表**（ReviewModeView） | `AudioFileListBox` | NavigationView → SplitView → Panel → ReviewModeView 深处 | `InlineListContextMenuController`（非 Popup 内嵌 Border） | ✅ 可用 |
| **B. 字幕列表**（ReviewModeView） | `SubtitleCueListBox` | 同 A，NavigationView → SplitView 深处 | 无右键菜单（仅双击跳转） | ⚠️ 未实现 |
| **C. Media Studio 会话/消息** | `SessionListBox` / `Border` | 独立 Window（`MediaStudioWindow`）或嵌入 NavigationView（`MediaStudioView`） | 原生 XAML 声明式 `ContextMenu` | ✅ 可用（Window 模式）/ ⚠️ 待验证（嵌入模式） |

### 1.2 核心问题回顾

根据《右键菜单在avaloniaui和FluentAvalonia中的特殊定义.md》的 10 组实验，**原生 `ContextMenu`（Popup 机制）在 FluentAvalonia `NavigationView` → `SplitView.Content` 内部完全不渲染**：

- 实验 8（最纯粹原生方案）：`ContextRequested` 确实触发且选中成功，但菜单 Popup 不可见
- 实验 6/7：`ContextMenu.Open()` 返回 `isOpen=True` 但 Popup 不渲染
- 实验 10：即使 `PlacementTarget = TopLevel` 也无效

这不是代码写法问题，而是 **Popup 层在 SplitView 内部的渲染机制限制**。

### 1.3 已验证可用方案

`InlineListContextMenuController<TItem>` 采用非 Popup 设计：

- 在 `ListBox` 的同级 `Grid` 中放置一个 `Border`（`ZIndex=1000`）
- 右键时通过 `TranslateTransform` 定位到点击坐标附近
- 用 `IsVisible` 控制显隐
- 用 `ClipToBounds=False` 允许面板溢出列表边界

## 三、现阶段测试覆盖评估

### 3.1 已覆盖的测试维度

| 维度 | 覆盖情况 | 说明 |
|------|---------|------|
| 音频列表右键弹出 | ✅ | InlinePanel 方案在生产中使用 |
| 右键目标项正确性 | ✅ | `SelectHitItemThenOpen` + `ResolveHitItem` |
| 菜单关闭路径 | ✅ 部分 | Esc / 外部点击 / 选择变化 / 菜单操作均有 |
| 日志审计 | ✅ | `AuditUiEvent` 覆盖 show/hide/click |
| CrashLogger 打点 | ⚠️ 不足 | 仅 `MenuShown` 有 Breadcrumb，关闭/失败路径无 |
| 字幕列表右键 | ❌ 未实现 | 仅双击跳转 |
| MediaStudioView 嵌入模式右键 | ❓ 未验证 | 使用原生 ContextMenu，但处于 NavigationView 内 |
| 边界条件：空列表右键 | ❓ 未明确 | controller 有 `selected == null` 返回逻辑 |
| 边界条件：快速连续右键 | ❓ 未明确 | `_pendingRightClickOpen` 状态机可能被竞争 |
| 键盘触发上下文菜单 | ❌ | InlinePanel 方案未处理 `ContextRequested`（键盘） |
| 高 DPI / 缩放场景 | ❓ 未测试 | 坐标 Clamp 逻辑依赖 `Bounds` |

### 3.2 测试缺口分析

**P0 缺口（直接影响用户体验）：**

1. 字幕列表 (`SubtitleCueListBox`) 无右键菜单 — 用户需要"右键跳转并播放"能力
2. MediaStudioView 嵌入模式的原生 ContextMenu 是否可见 — 可能重复踩 SplitView Popup 坑

**P1 缺口（稳定性/诊断）：**

3. CrashLogger Breadcrumb 在菜单关闭路径和失败路径缺少打点
4. 快速连续右键的状态一致性验证
5. 空列表/无选中项时的边界行为验证
6. `ContextMenuForceRegeneration` 配置对右键操作的实际影响验证

**P2 缺口（健壮性）：**

7. 键盘触发上下文菜单（Menu 键 / Shift+F10）
8. 高 DPI / 系统缩放下的坐标偏移
9. 窗口缩放后 ListBox Bounds 变化对 Clamp 的影响

## 四、待做验证实验（按优先级排序）

### 实验 A：MediaStudioView 嵌入模式 ContextMenu 可见性验证

**目的**：确认 `MediaStudioView`（NavigationView 内嵌 UserControl）的原生 `ContextMenu` 是否可见

**方法**：
1. 切换到 Media Studio 页面（嵌入模式，非独立窗口）
2. 创建一个会话并发送消息
3. 在 SessionListBox 中右键
4. 在消息 Border 上右键
5. 观察 ContextMenu 是否弹出

**预期**：
- 若 ContextMenu 不可见，说明 NavigationView 内所有 Popup 均受影响，需统一 InlinePanel 方案
- 若 ContextMenu 可见，说明问题可能与 ReviewModeView 的特定布局嵌套有关

**日志打点建议**：
```csharp
CrashLogger.AddBreadcrumb("MediaStudioView ContextMenu test: session-list right-click");
```

### 实验 B：SplitView ClipToBounds 排查

**目的**：确认 NavigationView 内部 SplitView 是否对子树设置了 `ClipToBounds=True` 导致 Popup 被裁剪

**方法**（原生诊断，不引入外部工具）：
1. 在 `ReviewModeView.AttachedToVisualTree` 回调中遍历视觉树到 MainWindow
2. 输出每一级的 `ClipToBounds` 值
3. 记录到 Breadcrumb 或 Trace

**代码草案**：
```csharp
// 在 AttachedToVisualTree 中添加（仅诊断用）
Visual? v = this;
var sb = new System.Text.StringBuilder("VisualTree ClipToBounds:\n");
while (v != null)
{
    if (v is Control c)
        sb.AppendLine($"  {v.GetType().Name} ClipToBounds={c.ClipToBounds}");
    v = v.GetVisualParent() as Visual;
}
CrashLogger.AddBreadcrumb(sb.ToString());
```

**后续动作**：
- 若发现某层 `ClipToBounds=True`，尝试在运行时将其设为 `False`
- 再次测试原生 `ContextMenu` 是否可见

### 实验 C：手动 `new Popup()` + `PlacementTarget = MainWindow`

**目的**：完全绕开 `ContextMenu` 自带的 Popup 管理，手工创建 Popup 并挂到 Window 层

**方法**：
```csharp
var popup = new Popup
{
    PlacementTarget = TopLevel.GetTopLevel(this) as Control,
    Placement = PlacementMode.Pointer,
    IsLightDismissEnabled = true,
    Child = new Border
    {
        Background = Brushes.White,
        Child = new TextBlock { Text = "测试菜单" }
    }
};
popup.Open();
```

**验证点**：
- Popup 是否在 MainWindow 层渲染可见
- 与 `ContextMenu.Open()` 返回 `isOpen=True` 但不渲染的行为对比

### 实验 D：FAMenuFlyout 替代原生 ContextMenu

**目的**：测试 FluentAvalonia 的 `FAMenuFlyout` 是否有不同的 Popup 层级处理

**方法**：
```csharp
var flyout = new FAMenuFlyout();
flyout.Items.Add(new MenuFlyoutItem { Text = "跳转并播放" });
flyout.ShowAt(subtitleListBox, true); // 或 ShowAt(topLevel, point)
```

**分析点**：
- FAMenuFlyout 内部是否使用与 ContextMenu 不同的 Popup/Overlay 层
- 是否能逃脱 SplitView 的裁剪

### 实验 E：字幕列表快速扩展 InlinePanel 方案

**目的**：若实验 B/C/D 均失败，为字幕列表复用 `InlineListContextMenuController<SubtitleCue>`

**设计思路**：
1. 在 `SubtitleCueListBox` 同级 `Grid` 中添加 `Border#SubtitleInlineMenuPanel`
2. 实例化 `InlineListContextMenuController<SubtitleCue>`
3. 菜单项："跳转并播放"、"复制字幕文本"
4. 选中模式设为 `SelectHitItemThenOpen`

**Breadcrumb 打点**：
```csharp
CrashLogger.AddBreadcrumb($"SubtitleInlineMenu shown: {cue.RangeText}");
CrashLogger.AddBreadcrumb($"SubtitleInlineMenu hide: reason={reason}");

```

### 实验 F：快速连续右键状态一致性

**目的**：验证 `InlineListContextMenuController` 在快速操作下不会出现状态错乱

**手工测试步骤**：
1. 选中第一个音频文件
2. 右键 → 菜单弹出 → 不点击菜单项
3. 立即在第二个文件上右键
4. 验证菜单目标切换到第二个文件
5. 重复 10 次，每次间隔 < 500ms
6. 验证 `CurrentItem` 与 `DataContext` 始终一致

**日志验证**：审查 `AuditUiEvent` 日志序列，确认无 show/hide 事件丢失

### 实验 G：键盘触发上下文菜单

**目的**：测试 Menu 键 / Shift+F10 能否触发 InlinePanel 菜单

**现状分析**：
- `InlineListContextMenuController` 仅监听 `PointerPressed/Released`
- 未注册 `ContextRequested` 事件
- 键盘用户无法打开右键菜单

**扩展方案**：
```csharp
// 在 controller 中新增
public void HandleContextRequested(ContextRequestedEventArgs e)
{
    var selected = _selectedItemProvider();
    if (selected == null) return;

    CurrentItem = selected;
    // 键盘触发时使用选中项中心位置
    _lastRightPressPosition = EstimateSelectedItemCenter();
    _lastRightReleasePosition = _lastRightPressPosition;

    if (TryShowInternal(selected, out var shownAt))
    {
        e.Handled = true;
        MenuShown?.Invoke(selected, shownAt);
    }
}
```

## 五、CrashLogger 打点增强建议


| 2 | ReviewModeView IsVisible=False 时事件注册时机 | ⚠️ | 实验 4 日志显示 Bounds=0 时注册可能无效 |
| 3 | InlineMenuPanel 在窗口缩放后的位置偏移 | ❓ | TranslateTransform 使用绝对坐标 |
| 4 | 触摸屏长按触发右键 | ❓ | Avalonia 的 `ContextRequested` 是否覆盖触摸长按 |
| 5 | 菜单按钮与 ListBox 选中态的视觉冲突 | ✅ | 已有 `ListBoxItem:pressed` 样式覆盖 |
| 6 | 多个 InlinePanel 同时可见 | ❓ | 音频列表和字幕列表各有一个时，需确保互斥 |
| 7 | DataContext 生命周期 | ⚠️ | InlineMenuPanel.DataContext 被设为选中项，需确认页面切换时不持有过期引用 |
| 8 | ContextMenuForceRegeneration 配置热更新 | ❓ | 修改配置后，下次右键操作是否立即生效 |

## 九、结论

1. **现阶段测试对音频列表右键场景覆盖较好**（弹出、目标、关闭、审计），但缺少边界条件和稳定性压力测试。
2. **字幕列表右键是最大功能缺口**，需优先推进。
3. **MediaStudioView 嵌入模式是最大未知风险**，需先做实验 A 确认原生 ContextMenu 是否可用。
4. **CrashLogger 打点有补充空间**，关闭路径和失败路径需要 Breadcrumb。
5. **根因排查（实验 B/C/D）值得尝试**，如果能找到 Popup 失效的确切原因，可能一次性解决所有 Popup 类菜单问题，消除对 InlinePanel 的依赖。
6. **键盘上下文菜单、触摸长按、页面切换残留**是容易被忽略的边缘场景。
